# 要約フォーマット仕様

## Frontmatter 構造

mdファイルのfrontmatterは以下の形式に従う：

```yaml
---
title: "記事タイトル"
source: "https://example.com/article"
author:
  - "[[Author Name]]"
published: YYYY-MM-DD
created: YYYY-MM-DD
description: "記事の概要（100-200文字）"
tags:
  - "clippings"
  - "tag1"
  - "tag2"
---
```

### フィールド説明

| フィールド | 必須 | 説明 |
|-----------|------|------|
| title | Yes | 記事/動画のタイトル |
| source | Yes | 元のURL |
| author | No | 著者名（`[[Name]]` 形式推奨） |
| published | No | 公開日（YYYY-MM-DD形式） |
| created | Yes | ファイル作成日 |
| description | No | 概要（要約スキルで補完） |
| tags | No | タグ配列（要約スキルで補完） |

## Web記事用要約フォーマット

```markdown
## 概要

[1-2段落で記事の全体像を説明]

## 主要なトピック

### トピック1

- ポイント1
- ポイント2
- ポイント3

### トピック2

- ポイント1
- ポイント2

## 重要な事実・データ

- **数値データ1**: 具体的な数字や統計
- **数値データ2**: 引用可能な主張
- **技術詳細**: 技術的な詳細情報

## 結論・示唆

### 著者の結論

[著者が述べている結論]

### 実践的な示唆

- 示唆1
- 示唆2
- 示唆3

## 制限事項・注意点

[元ソースに記載されている制限事項や注意点]

---

*Source: [記事タイトル](URL)*
```

## YouTube動画用要約フォーマット

```markdown
## 動画概要

- **チャンネル**: [[Channel Name]]
- **公開日**: YYYY-MM-DD
- **長さ**: HH:MM:SS

## タイムスタンプ付き要約

### [00:00] イントロダクション

- 動画の導入部分の要点
- 話者の自己紹介（該当する場合）

### [05:30] メインセクション1

- ポイント1
- ポイント2
- 引用: "重要な発言"

### [15:00] メインセクション2

- ポイント1
- ポイント2

### [25:00] 実践・デモ（該当する場合）

- デモの内容
- 重要な手順

### [35:00] まとめ・結論

- 結論のポイント

## キーメッセージ

1. **メッセージ1**: 説明
2. **メッセージ2**: 説明
3. **メッセージ3**: 説明

## 結論・示唆

### 話者の結論

[話者が述べている結論]

### 実践的な示唆

- [ ] アクションアイテム1
- [ ] アクションアイテム2
- [ ] アクションアイテム3

## 関連リソース

動画内で言及されたリソース：

- [リソース1](URL)
- [リソース2](URL)

---

*Source: [動画タイトル](URL)*
```

## タグ生成ガイドライン

要約時にtagsを生成する際の基準：

### カテゴリタグ

- `clippings`: Web記事のクリッピング
- `youtube`: YouTube動画
- `podcast`: ポッドキャスト

### トピックタグ（例）

| 分野 | タグ例 |
|------|--------|
| AI/ML | `AI`, `Machine Learning`, `LLM`, `GPT` |
| 開発 | `Programming`, `TypeScript`, `Python`, `Go` |
| クラウド | `AWS`, `GCP`, `Azure`, `Kubernetes` |
| ビジネス | `Strategy`, `Leadership`, `Management` |
| キャリア | `Career`, `Productivity`, `Learning` |

### タグ数の目安

- 最低: 3個
- 推奨: 5-7個
- 最大: 10個

## 品質チェックリスト

要約生成後に確認すべき項目：

### 完全性

- [ ] 重要なポイントが漏れていないか
- [ ] 元の構造が反映されているか
- [ ] 数値データが正確に引用されているか

### 明瞭性

- [ ] 読みやすい構造になっているか
- [ ] 専門用語に説明が必要でないか
- [ ] 文章が簡潔か

### 正確性

- [ ] 事実が正確に伝わっているか
- [ ] 著者の意図が歪められていないか
- [ ] 引用が正確か

### メタデータ

- [ ] author が補完されているか
- [ ] published が補完されているか
- [ ] description が適切か
- [ ] tags が適切か

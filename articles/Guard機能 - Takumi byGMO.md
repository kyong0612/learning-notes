## 概要

**Takumi Guard** は、GMO Flatt Security が提供するパッケージレジストリ向けのセキュリティプロキシサービスである。`.npmrc` などの設定ファイルに1行追加するだけで、`npm install` 等のパッケージインストール時に悪意あるパッケージを **インストール前にブロック** する。さらに、過去にインストールしたパッケージが後から危険と判明した場合にも **フォローアップ通知** を行い、すり抜けた脅威も見逃さない仕組みを提供する。

現在 **npm** および **PyPI** に対応済みで、**crates.io**（Rust）や **RubyGems**（Ruby）にも順次対応予定。契約や決済は不要で、すべて無料で利用できる。

### HOW IT WORKS — 設定1行でブロック、インストール後も見逃さない

Takumi Guardの動作は以下の3ステップで構成される。

#### ステップ1: レジストリを設定

`.npmrc` に1行追加するだけで、すべてのインストールがTakumi Guardのプロキシを経由する。

```
registry=https://npm.flatt.tech
```

#### ステップ2: マルウェアをブロック

ブロックリストにあるパッケージは、インストールする前に停止される。

```terminal
$ npm install @panda-guard/test-malicious
[guard] BLOCKED: credential theft
```

#### ステップ3: フォローアップ通知

過去にインストールしたパッケージが後から危険と判明した場合も通知。すり抜けた脅威も見逃さない。

### BEHIND THE SCENES — すべてのパッケージを監視

Takumiはレジストリに公開されるすべてのパッケージを継続的に監視し、脅威を検出している。検出結果の例として以下が挙げられている。

- **インストールスクリプトの悪用**
- **使い捨てメールからの公開**
- **typosquatting（名前偽装）**
- **既知の悪意あるメンテナーの検出**

## 重要な事実・データ

- **導入の容易さ**: 設定ファイルに1行追加するだけで導入完了
- **対応パッケージマネージャ**: npm, pnpm, Yarn v1, Yarn v2+, bun, deno
- **価格**: 完全無料（契約・決済不要）
- **プロキシ方式**: レジストリプロキシとして動作し、パッケージインストール前にブロック
- **事後検知**: インストール済みパッケージが後から危険と判明した場合のフォローアップ通知あり
- **検出手法**: インストールスクリプト悪用、使い捨てメール公開、typosquatting、既知の悪意あるメンテナー検出

## 結論・示唆

### サービスの特徴

Takumi Guardは、近年急増するソフトウェアサプライチェーン攻撃（特にnpmエコシステムを標的としたもの）に対して、**開発ワークフローを変えることなく**防御層を追加できるツールである。AIエージェントを悪用した新たな攻撃手法にも対応しており、現代の開発環境における脅威に適合している。

### 実践的な示唆

- `.npmrc` に `registry=https://npm.flatt.tech` を追加するだけで即座に保護が有効になる
- 個人開発者からチーム開発まで対応可能
- 特にAIコーディングアシスタントを利用する開発環境では、AIエージェント経由の攻撃を防ぐ意味でも導入が推奨される
- 無料で利用できるため、導入障壁が非常に低い

## 概要

AI音声入力アプリ「Typeless」のセキュリティリスクがSNS上で話題となり、デジライズCEOのチャエン氏（茶圓将裕）が直接Typeless社に問い合わせ、同社エンジニアにも確認した上で正式回答を共有した投稿。2026年2月上旬にリバースエンジニアリング結果がX上で拡散されたことをきっかけに、広範な議論が行われた。

## Typelessとは

- スタンフォード大学卒業生が設立した音声入力アプリ（**Simply CA LLC**、カリフォルニア州）
- macOS / Windows / iOS / Android対応
- AIを使って話し言葉をリアルタイムでテキストに変換
- フィラー言葉の除去、文章整形、翻訳機能を搭載
- CEO: Huang Song

## 公式の主張

Typeless公式プライバシーポリシーでは以下を掲げている：

- **Zero data retention**（データ保持なし）：音声データはリアルタイム処理後に即時削除
- モデルトレーニングには使用しない
- 業界標準の暗号化とアクセス制御を採用
- OpenAI等の第三者AIパートナーとのゼロ保持契約を保証

## リバースエンジニアリングで判明した主な懸念点

2026年2月上旬、エンジニアの**げれげれ氏**（[@medmuspg](https://x.com/medmuspg)）がリバースエンジニアリング結果を共有し、以下の深刻な問題が指摘された。

### 1. クラウド処理への完全依存とデータ送信リスク

- 音声認識は**100%クラウドベース**（ローカルモデルなし）
- すべての音声データが**AWSサーバー（米国オハイオ地域）**に送信
- 音声以外のコンテキスト情報（使用アプリ名、画面テキスト）も送信される
- 通信中の傍受やサーバー侵害のリスクあり

### 2. 過剰なデータ収集（キーロガー級）

アプリは音声入力以外にも以下のデータを収集可能：

| 収集データ | 詳細 |
|---|---|
| 閲覧URL | Gmail、Google Docsを含む |
| アプリ情報 | フォーカス中のアプリ名・ウィンドウタイトル |
| 画面テキスト | アクセシビリティAPI経由 |
| クリップボード | コピー内容 |
| キーボード入力 | CGEventTap使用による監視 |
| ブラウザDOM | ブラウザ内の要素情報 |

- これらはローカルDB（`typeless.db`）に**平文で保存**
- 音声ファイルも削除されず残存
- 技術的に**キーロガーと同等**との指摘

### 3. 権限の過剰要求

音声入力ツールとして必要以上に広範な権限を要求：

- マイク
- 画面録画
- カメラ
- Bluetooth
- アクセシビリティ

### 4. 運営企業の不透明性

- 利用規約に**法人名の記載なし**
- 所在地は「サンフランシスコ郡、CA」のみ
- **WHOIS非公開**
- SOC2やISO27001などの**セキュリティ監査記載なし**
- 連絡先はメールのみ
- リバースエンジニアリングを禁止する条項あり

## 重要な概念の区別

> **「ゼロ保持」≠「外部送信なし」**

- サーバーに保存しないことと、外部にデータを送信しないことは別の概念
- 医療情報、営業秘密など機密性の高い情報は、**処理中にクラウドへ送信される時点でリスクが発生**する

## ユーザー・専門家の反応

2026年2月上旬からX上で懸念投稿が急増し、意見は大きく3つに分かれた：

1. **企業・機密利用の非推奨** — 金融・医療・法律分野で特にリスク大、コンプライアンス違反の可能性
2. **利便性 vs リスクのトレードオフ** — 「便利だがクラウド処理の代償」「プライバシー重視なら避けるべき」
3. **深刻なリスク指摘と即時アンインストール推奨** — 「スパイウェアに近い」「キーロガー同等」との声が多数

全体として**ネガティブな意見が優勢**。

## Typeless側の対応

- セキュリティ懸念の拡散後、公式ウェブサイトに**「信用センター」（Trust Center）**メニューを新設
- 迅速な対応として一定の評価を受けている

## 利用時の推奨事項

| 対策 | 内容 |
|---|---|
| 機密情報の回避 | パスワード、財務データ、個人情報を扱うアプリ（Gmailなど）では使用しない |
| ネットワーク監視 | Little Snitchなどのツールで通信をチェック |
| 権限の最小化 | 不要な権限（カメラ等）は拒否 |
| ローカルデータ管理 | DBや音声ファイルを定期的に削除、バックアップ時は暗号化 |
| ポリシー確認 | アプリ更新時にプライバシーポリシーの変更をチェック |

## 結論

Typelessは生産性を高める便利なツールだが、**クラウド依存の処理構造**と**過剰なデータ収集範囲**がプライバシーを脅かす可能性が高い。特に機密業務での使用は避け、軽いメモ用途に限定するのが現時点での安全な利用方針とされている。公式のトラストセンター設置は前向きな動きだが、SOC2等の第三者監査やデータ収集範囲の限定など、具体的な改善が求められている。

## 概要

Cloud Runの使用時に適用される使用量割り当てと上限について説明するドキュメント。Cloud Runのリソース数には上限が適用され、[Cloud Run Admin API](https://docs.cloud.google.com/run/docs/reference/rest?hl=ja)の呼び出しレートにも影響を与える。

### 直接的な上限がないもの

- デプロイできる**コンテナイメージのサイズ**
- Cloud Runサービスによって処理される**同時リクエストの数**

## Cloud Run のリソースの上限

### サービス・ジョブ関連

| リソース | 範囲 | 上限 | 引き上げ可能 |
|---------|------|------|:------------:|
| サービス | プロジェクト×リージョン | 1,000 | × |
| ジョブ | プロジェクト×リージョン | 1,000 | × |
| 実行中のジョブ | プロジェクト×リージョン | 1,000 | × |
| リビジョン | サービスごと | 1,000 | × |
| トラフィック処理リビジョン | プロジェクト×リージョン | 4,000 | × |
| リビジョンタグ | プロジェクト×リージョン | 2,000 | × |
| 完了ジョブ実行の保持 | ジョブごと | 1,000 | × |

### ジョブ実行タスク関連

| リソース | 範囲 | 上限 | 引き上げ可能 |
|---------|------|------|:------------:|
| 並列実行タスク数 | ジョブ実行ごと | リージョン・CPU・メモリ構成により異なる | ○ |
| タスク最大タイムアウト | ジョブ実行ごと | **168時間（7日）** ※24時間超はプレビュー版 | × |
| 1ジョブあたりの最大タスク数 | ジョブ実行ごと | 10,000 | × |
| タスク再試行回数 | ジョブ実行ごと | 10 | × |

### コンテナインスタンス関連

| リソース | 範囲 | 上限 | 引き上げ可能 |
|---------|------|------|:------------:|
| ダイレクトVPC下り利用インスタンス | リビジョン×リージョン | 100〜200（リージョンにより異なる） | ○ |
| 起動タイムアウト | インスタンスごと | **4分** | × |
| 最大メモリ | インスタンスごと | **32 GiB** | × |
| 最大vCPU | インスタンスごと | **8** | × |
| 書き込み可能メモリ内ファイルシステム | インスタンスごと | 32 GiB（メモリにより制限） | × |

### プロジェクト・リージョン単位のリソース

| リソース | 上限 | 引き上げ可能 |
|---------|------|:------------:|
| CPU合計（ミリvCPU/分） | リージョンにより異なる | ○ |
| メモリ合計（バイト/分） | リージョンにより異なる | ○ |
| ゾーン冗長性なしGPUインスタンス | 0（新規プロジェクトは3付与） | ○ |
| ゾーン冗長性保証付きGPUインスタンス | 0 | ○ |

### その他の上限

| リソース | 範囲 | 上限 |
|---------|------|------|
| 環境変数数 | ジョブ/サービスごと | 1,000 |
| コマンド引数数 | ジョブ/サービスごと | 1,000 |
| 環境変数の長さ | 変数ごと | 32 KB |
| SSL証明書 | トップドメイン/週 | 50 |
| 重複SSL証明書 | トップドメイン/週 | 5 |
| アクセストークン生成 | インスタンス/秒 | 50 |
| IDトークン生成 | インスタンス/秒 | 50 |
| 同時オープンファイル数 | インスタンスごと | 25,000 |

## Cloud Run のネットワークの上限

### リクエストの上限

| 説明 | 上限 |
|------|------|
| インスタンスあたりの同時リクエスト | **1,000** |
| HTTP/2接続あたりの同時ストリーム | 100 |
| リクエストタイムアウト | **60分** |
| HTTP/1リクエストサイズ | HTTP/1サーバー: 32 MiB、HTTP/2サーバー: 無制限 |
| HTTP/1レスポンスサイズ | chunkedエンコーディング未使用時: 32 MiB |
| インスタンスあたりのオープン接続数（下り種別ごと） | 50,000（ダイレクトVPC下り除く） |
| アウトバウンド接続（インスタンス/秒） | 700（ダイレクトVPC下りのVPC宛は制限なし） |
| アウトバウンドDNS解決（インスタンス/秒） | 1,000 |
| HTTP/1インバウンドリクエスト（インスタンス/秒） | 800（HTTP/2は適用外） |

### 帯域幅の上限

| 説明 | 上限 |
|------|------|
| ダイレクトVPC経由下り | **1 Gbps** |
| それ以外の上り/下り合計 | **600 Mbps** |
| パケットレート（ダイレクトVPC下り除く） | 64,000パケット/秒 |

> **注意**: いずれかの上限に達すると、Cloud Runインスタンスの帯域幅が制限される

## Cloud Run Admin API のレート上限

Admin APIの上限は**デプロイされたCloud Runサービスへのリクエストには適用されない**。

| 割り当て | 上限 | 引き上げ可能 | スコープ |
|---------|------|:------------:|----------|
| API読み取りリクエスト | 3,000/60秒 | ○ | プロジェクト×リージョン |
| API書き込みリクエスト | 180/60秒 | ○ | プロジェクト×リージョン |
| ジョブ実行 | 180/60秒 | ○ | プロジェクト×リージョン |

## 割り当てを増やす方法

### 手順（Google Cloud コンソール）

1. **[IAM と管理] > [割り当てとシステム上限]** ページに移動
2. 更新する割り当て値を見つけ、チェックボックスをオン
3. **[編集]** をクリック
4. **[新しい値]** フィールドに必要な割り当て値を入力
5. ダイアログの上限を超える場合は **[割り当ての増加を申し込む]** を選択
6. **[リクエストを送信]** をクリック

### 重要なポイント

- 割り当て調整リクエストは**審査の対象**
- リクエストの受領確認メールが届き、審査後に結果通知メールが届く
- コンソールから調整できない場合は[Cloud カスタマーケア](https://docs.cloud.google.com/support?hl=ja)に連絡

### 複数リクエストのまとめ方

- 複数の割り当て引き上げを一括リクエスト可能
- **一括処理は審査時間が長くなる可能性あり**
- 審査時間短縮のため、**プロダクトとエリア別にグループ化**することを推奨
  - 例: ネットワーキングとCompute Engine VMは別々にリクエスト

## 特記事項

### リージョン別割り当ての特徴

- CPU・メモリ・並列処理数などの割り当ては**リージョンにより異なる**
- 大容量リージョンでは高く、新規開設リージョンでは低い傾向
- [割り当てとシステム上限ページ](https://console.cloud.google.com/iam-admin/quotas?hl=ja)で確認可能

### GPU関連

- L4 GPUを使用する新規プロジェクトには**3つのNvidiaL4GpuAllocNoZonalRedundancyPerProjectRegion**が自動付与
- ピーク時のGPU使用量が割り当てに近付かない場合、**割り当てが減らされる可能性あり**

### タイムアウト関連

- ジョブタスクの24時間超タイムアウトは**プレビュー版**で利用可能
- サービスの起動タイムアウトは4分固定

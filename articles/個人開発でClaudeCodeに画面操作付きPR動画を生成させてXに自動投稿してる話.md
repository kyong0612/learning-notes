## 概要

個人開発で教員向けAI校務支援サービス（[ai-teacher-support.com](https://ai-teacher-support.com/)）を運営する著者が、PR動画の作成・投稿を自動化したパイプラインを紹介する記事。AIによる開発で新機能の追加が容易になった一方、手動でのPR動画作成が追いつかず、機能がユーザーに認知されないという課題を解決するために構築された。

人間が行うのは**台本と動画の品質チェックのみ**で、それ以外はすべて自動化されている。

## 主要なトピック

### 全体アーキテクチャ

パイプラインは3段構成で動作する：

```
1. Playwright デモテスト → ブラウザ操作を自動実行し .webm 動画を録画
2. Remotion コンポジション → 録画をブランディングされたプロモ動画に合成
3. 出力 MP4 (1920x1080, H.264) → SNS投稿に使用
```

Playwrightが録画した操作動画を「素材」として、Remotionが「完成動画」に仕上げるという2段構成がポイント。

### 生成手順

1. PlanモードでClaude Codeに「Remotion + Playwrightで〇〇機能のPR動画作成」を依頼
2. 生成された台本を確認
3. 問題なければPR動画の生成を依頼
4. 生成後の品質確認
5. 気になる点があれば指摘・修正依頼
6. 問題なければ push → GitHub Actions が台本の内容を読み取り、PR動画を添付して X に自動投稿

### Step 1: Playwrightでデモ操作を自動録画

通常のE2Eテストとは別に、**デモ録画専用**の設定ファイルを用意。`video.mode: 'on'` と `size: { width: 1920, height: 1080 }` を設定し、テスト実行中のブラウザ画面を1080pのwebm動画として自動保存する。

デモテストの特徴：
- APIをモックして**理想的なデータ**を返す
- ユーザーに見せたい操作手順を「台本」として記述

| 工夫 | 理由 |
|------|------|
| `pressSequentially` で1文字ずつ入力 | タイピングの様子をリアルに見せる |
| `mouse.move` で段階的にカーソル移動 | マウスの動きをスムーズに見せる |
| APIモックに意図的な遅延（600-800ms） | AI処理のローディング状態を自然に見せる |
| Next.jsエラーオーバーレイの非表示 | 開発サーバーのオーバーレイが映り込むのを防止 |
| キャッチオールAPIモック | 未定義のAPIリクエストで画面が壊れるのを防止 |
| `page.video()?.saveAs()` | 録画をRemotionの素材パスに直接保存 |

### Step 2: Remotionでプロモーション動画に仕上げる

各動画は統一された**4セクション構成**：

| セクション | 時間 | 内容 |
|-----------|------|------|
| Hook | 0-2秒 | ロゴ + キャッチコピー（Springアニメーション付き） |
| Problem | 2-4秒 | 課題提示（フェードアップアニメーション） |
| Solution Demo | 4-12秒 | Playwrightで録画した操作動画を再生 |
| CTA | 12-15秒 | 「まずは無料で始めよう」+ ボタン風UI + URL |

**可変速再生**にも対応しており、シーンごとに再生速度を変更可能。AI処理中は3倍速で早送り、ユーザーに見せたい入力シーンは0.8倍速でゆっくり再生といった制御ができる。

再利用可能なコンポーネント群（`AnimatedVideo`, `HookScene`, `CtaScene`, `SceneContainer` 等）を用意しており、新機能のデモ動画を作る際はコンポジションファイルを1つ追加するだけで済む。

ブランドカラーとSpringアニメーションの設定は `theme.ts` で一元管理。

### Step 3: バッチレンダリング

全9コンポジションを一括でMP4にレンダリングするスクリプトを用意。実行は以下の3コマンド：

```bash
npm run test:demo              # デモ動画を録画（Playwright）
bash scripts/preserve-demo-videos.sh  # 録画素材を保存
npm run remotion:render:all    # プロモ動画をレンダリング（Remotion）
```

個別レンダリングも可能（`npm run remotion:render:report` 等）。

### X（Twitter）自動投稿との連携

別リポジトリ（auto-postリポジトリ）で構築したXへの自動投稿バッチと連携。push すると GitHub Actions が台本の内容を読み取り、PR動画を添付して X に自動投稿する。

```
push → GitHub Actions トリガー → 台本の内容を読み取り → PR動画付きで X に自動投稿
```

## 重要な事実・データ

- **サイト流入数**: 動画付き投稿により、リンクからサイトへ流入する人数が**約1.8〜2倍に増加**
- **Xのview数**: 投稿のview数自体に大きな変化はなし
- **動画仕様**: 1920x1080, H.264, 30fps, 15秒（450フレーム）
- **コンポジション数**: 全9コンポジション
- **人間の作業**: 台本と動画の品質チェックのみ

## 結論・示唆

### 著者の結論

- 動画付きの投稿はview数には直結しなくても、興味を持った人が実際にサービスを見に来る確率を押し上げる効果がある
- 一度パイプラインを組めば、AIにPR動画作成を依頼するだけでXに自動投稿される体験はかなり快適
- テストを書くついでにデモ素材も手に入る、「一石二鳥」の仕組み

### 実践的な示唆

- Playwright + Remotion の組み合わせは、個人開発でデモ動画作成が面倒な場合に有効
- E2Eテストとデモ録画を兼用することで効率化が可能
- APIモックを活用することで理想的なデモシナリオを再現できる
- 再利用可能なコンポーネント設計により、新機能追加時の動画作成コストを最小化できる

## 制限事項・注意点

- Xのview数自体への直接的な効果は確認されていない（流入数は増加）
- 将来的にクオリティが担保できれば台本確認すら不要な完全自動化を目指しているが、現時点では人間による品質チェックが必要

## 概要

2026年3月11日、クラウドセキュリティ企業 **Wiz** がGoogleによる買収を正式に完了し、**Google Cloud** の一部として新たな章を開始したことを発表した。Wiz CEO の **Assaf Rappaport** が、同社のミッション、買収プロセス中の成果、そして今後の方向性について述べている。

---

## AIがもたらすセキュリティの新時代

- クラウドがチームの開発速度を変革したように、**AIが再びイノベーションを加速**させている
- 生成AIはもはや実験的なものではなく、組織がアプリケーションを構築・出荷・スケールする方法の中核となっている
- アイデアから本番環境への移行が数分で行われる時代において、**セキュリティは進歩を加速させるもの**でなければならない
- Wizは、コード・クラウド・ランタイム全体の豊富なコンテキストを活用し、チームがAI搭載アプリケーションを最初からセキュアに構築できるようにする

> セキュリティリーダーは、スピードとセキュリティのどちらかを選ぶ必要はない。

---

## 買収プロセス中の主要な成果

買収手続き期間中も、Wizチームは開発を止めることなく、多くのマイルストーンを達成した。

### Wiz Researchの脆弱性発見

| 発見名 | 内容 |
|---|---|
| **Moltbook** | AIエージェント向けソーシャルネットワークの露出したデータベースから数百万のAPIキーが漏洩。「バイブコーディング」アプリのセキュリティリスクを浮き彫りに |
| **CodeBreach** | AWS Consoleを侵害し得るクリティカルなサプライチェーン脆弱性（AWS CodeBuild） |
| **RediShell** | Redisに13年間存在していたクリティカルなRCE脆弱性（CVSS 10.0）。クラウド環境の75%以上に影響 |
| **NVIDIAScape** | 共有AIインフラストラクチャを脅かすコンテナエスケープ脆弱性 |
| **Lovableとの連携** | バイブコーディングリーダーのプラットフォームを強化。**5組織に1組織がシステミックリスクに晒されている**ことを発見 |
| **Shai-Hulud / NX** | 高度に標的化されたサプライチェーン攻撃の発見と修復。数百の組織を保護 |

### ZeroDay.cloud

世界トップクラスの研究者が参加する初の試みとなるハッキングコンペティションを開催。基盤となるクラウド・AIツールにおいて**記録的な数のCVE**を発見した。

## 製品イノベーション

### Wiz AI Security Platform
- AI アプリケーション自体のセキュリティを確保
- AI使用状況の可視化、AIネイティブリスクの防止、ランタイムでのAIワークロード保護

### Wiz Exposure Management
- コードからクラウド、オンプレミスまでの脆弱性と攻撃対象面の管理を統合
- チームが真に重要なリスクに集中し、悪用可能なリスクをプロアクティブに除去

### AI Security Agents
- コード・クラウド・ランタイム全体の深いコンテキストを活用
- リスクの調査・優先順位付け・修復を**マシンスピード**で自動化

### WizOS
- ほぼゼロCVEのハード化されたコンテナベースイメージ
- 最初のコミットから信頼できる基盤を開発者に提供

## Google Cloud との統合戦略

### AIとマルチクラウドの強み

- Googleの最先端AI機能（**Gemini**）をWizプラットフォームに統合予定
- セキュリティチームに新たな「スーパーパワー」を提供

### マルチクラウド対応の継続

- **Fortune 100企業の大半**、**フロンティアAIラボの大半**、および世界最速成長のクラウドネイティブ企業と取引
- **AWS、Azure、GCP、OCI** すべてに対応を継続
- Googleへの加入により焦点が狭まるのではなく、**強化される**

### Google エコシステムの活用

| リソース | 提供価値 |
|---|---|
| **Googleのインフラストラクチャ** | スケーラビリティと信頼性 |
| **Mandiantの脅威インテリジェンス** | 高度な脅威検出能力 |
| **Google Unified Security Platform** | 統合セキュリティエコシステム |

## 重要な結論

1. **ミッションは不変**: すべての組織が構築・運用するすべてのものを保護する
2. **マルチクラウドは維持**: Google傘下でも、AWS・Azure・OCI対応を継続
3. **AI時代のセキュリティ**: セキュリティはイノベーションの障害ではなく、加速装置であるべき
4. **Google統合の利点**: Gemini AI、Mandiantの脅威インテリジェンス、Google Unified Security Platformとの連携により、より強力なセキュリティを提供
5. **顧客との信頼**: 日々の行動、製品、イノベーションのペースを通じて信頼を獲得し続ける

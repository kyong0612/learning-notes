## セキュリティにおけるネットワークの位置づけ

### 核心的な問題意識

AWSには多様なセキュリティアプローチが存在する：

- IAM権限
- セキュリティグループ
- WAF
- SSL/TLS暗号化
- RDS暗号化
- Secrets Manager

これらは並列に見えるが、実際には異なる目的を持つ。基本的な疑問は「どのサービスをどこに適用すべきか？」である。

### OSIモデルによる階層的理解

**解決策：OSIモデルの活用**

AWSサービスは異なるOSI階層で動作する：

- **セキュリティグループ**：レイヤー3-4（ネットワーク/トランスポート層）
- **IAM**：レイヤー7（アプリケーション層）

低レイヤーのフィルタリングは上位レイヤーの認証より前に順次実行される。例えは「熊が村に到着する前に山から出ないように止める」のようなもの。

## AWSネットワークセキュリティサービスの分類

### サービスカテゴリ

| カテゴリ | サービス | OSI層 |
|---------|---------|--------|
| **セグメント化** | VPC、サブネット、ルートテーブル | レイヤー3 |
| **ファイアウォール** | ネットワークACL（ステートレス）<br>セキュリティグループ（ステートフル）<br>AWS Network Firewall | レイヤー3-7 |
| **IP隠蔽** | NAT Gateway、NATインスタンス | レイヤー3 |
| **プロキシ/中継** | ALB、API Gateway | レイヤー7 |
| **暗号化** | MACsec、VPN、TLS/SSL、DNSSEC | 各層 |
| **攻撃防御** | AWS Shield（DDoS）<br>AWS WAF（アプリケーション攻撃） | レイヤー7 |
| **包括的サービス** | GuardDuty、Security Hub、Network Security Director | 複数層 |

### 重要な概念

1. **レイヤーの理解**
   - 各サービスが動作するOSI層を理解することが重要
   - 下位層でのフィルタリングが上位層の処理前に実行される

2. **ステートフル vs ステートレス**
   - **セキュリティグループ**：ステートフル（戻りトラフィックを自動許可）
   - **ネットワークACL**：ステートレス（明示的な双方向ルール必要）

## アーキテクチャの実例

### 公開Webアプリケーションの構成例

記事では以下の要素を含む構成を提示：

1. **サブネットのセグメント化**
   - パブリックサブネット
   - ファイアウォールサブネット
   - アプリケーションサブネット
   - データベースサブネット

2. **階層的フィルタリング**
   - ネットワークACLによる第一防御
   - セキュリティグループによる細かい制御

3. **高度なセキュリティ機能**
   - **AWS Network Firewall**：高度なアウトバウンド制御
   - **NAT Gateway**：IPマスキング
   - **ALB**：外部エントリーポイント
   - **TLS/SSL**：通信暗号化
   - **WAF & Shield**：攻撃緩和

### 設計の考え方

- 非機能要件に基づいた適切なサービス選択
- コスト考慮
- セキュリティレベルと運用コストのバランス

## まとめ

ネットワークセキュリティの理解には以下が必要：

1. **各サービスの動作階層の認識**
   - OSI参照モデルを基準とした理解
   - レイヤーごとのセキュリティ制御の違い

2. **適切な配置の判断**
   - 非機能要件に基づく選択
   - コスト効率の考慮
   - セキュリティとパフォーマンスのトレードオフ

3. **段階的な防御戦略**
   - 低レイヤーから高レイヤーへの順次フィルタリング
   - 深層防御（Defense in Depth）の実践

この体系的なアプローチにより、AWSにおけるネットワークセキュリティの全体像を理解し、適切なサービス選択と配置が可能になる。

## 概要

State of JavaScript 2025 のライブラリセクションは、JavaScript エコシステムにおける主要ライブラリの使用状況、満足度、トレンドを包括的に調査したものである。約13,000人の回答者のデータに基づき、**フロントエンドフレームワーク**、**メタフレームワーク**、**バックエンドフレームワーク**、**テストツール**、**ビルドツール**の5カテゴリにわたって分析が行われている。

最も注目すべきトレンドとして、**Vite**が数年連続で最も愛されたライブラリであり続け、使用率でもReactを追い抜いて第2位に浮上した。一方で**Next.js**は最も分極化したフレームワークであり、使用率では上位ながら不満の声も多い。

## 主要なトピック

### 1. ライブラリ全体の使用率とセンチメント

使用率トップ10は以下の通り：

| 順位 | ライブラリ | カテゴリ | 使用率 | 認知率 |
|------|-----------|---------|--------|--------|
| 1 | webpack | Build Tools | 86.4% | 11.5% |
| 2 | **Vite** | Build Tools | 84.4% | 12.8% |
| 3 | React | Front-end | 83.6% | 14.4% |
| 4 | Express | Back-end | 79.9% | 16.6% |
| 5 | Jest | Testing | 74.4% | 21% |
| 6 | Next.js | Meta-Framework | 58.6% | 39.6% |
| 7 | Storybook | Testing | 56.1% | 30.5% |
| 8 | esbuild | Build Tools | 53.8% | 32.5% |
| 9 | Vitest | Testing | 53.8% | 30.8% |
| 10 | tsc CLI | Build Tools | 52.7% | 14.1% |

**キーインサイト：**
- **Vite**はReactを追い抜き使用率2位に。来年にはwebpackを抜いてトップになる可能性がある
- **Next.js**は13番目に愛されている一方、5番目に嫌われており、最多のコメント数（69件）を集めた最も分極化したフレームワーク
- 使用率の平均は27%、ポジティブ評価の平均は21%、ネガティブ評価の平均は19%

### 2. 経時変化トレンド（Changes Over Time）

JavaScriptライブラリには「ブーメラン」パターンが見られる：初期のポジティブな評価で右方向に進み、使用率増加で上方向に伸び、その後実際の問題に直面してネガティブに転じる（左方向に進む）。

**注目すべきトレンド：**
- **Vue.js** — 右方向に移動（ポジティブ評価を維持）しており注目に値する
- **Astro** — ほぼ垂直に上昇（使用率が増加しつつポジティブ評価を維持）
- **React** — 使用率は高いが、ポジティブ評価は年々低下（2016年の68.5%→2025年の31.0%）
- **Next.js** — 2025年に初めてネガティブ評価に転落（-3.08%）
- **Playwright** — 急速に成長し、2025年にポジティブ領域に到達（+42.97%）
- **Vitest** — 最速の成長を見せ、2025年にポジティブ領域に到達（+52.21%）
- **webpack** — 使用率は高いがネガティブ評価が拡大（-38.45%）
- **Gatsby** — 大幅にネガティブ方向に移動し衰退が顕著

## カテゴリ別詳細

### フロントエンドフレームワーク

**ハイライト: Solid**
- 使用率は10%にとどまるが、**5年連続で最高の満足度**を記録
- 作者のRyan Carniato はシグナルなどのコンセプトの普及に大きく貢献

**主要な発見：**
- 使用率ランキングは前年からほぼ変動なし（Alpine.jsとHTMXの入れ替えのみ）
- 満足度もQwikの低下を除いて安定
- 平均使用フレームワーク数は**2.6個**（月単位でフレームワークを乗り換えるというイメージは実態に合わない）
- フロントエンド全体の幸福度は5点満点中**3.7**（緩やかな低下傾向だが安定）

**ペインポイント：**
1. Reactに関する問題（601件）
2. 過度な複雑さ（435件）
3. パフォーマンス（239件）
4. 状態管理（217件）
5. 選択肢の多さ（215件）

**注目の新規参入：** Ripple（フロントエンドフレームワークの新参者として最も注目）

### メタフレームワーク

**ハイライト: Astro**
- 他の静的サイトジェネレーターのベストプラクティスを取り入れ、シンプルかつパワフルなフレームワークを実現
- **Cloudflareと提携**し、今後の展開がさらに期待される
- 満足度でカテゴリ1位

**主要な発見：**
- **Next.js**は使用率で支配的だが、満足度が低下し続けている。リーダーのAstroとの満足度差は**39ポイント**
- **Astro**は数年で「ほぼ無名」から「本格的な競合」へと急成長
- 平均使用メタフレームワーク数は**1.7個**（1-2つのソリューションが支配的なカテゴリ）
- **TanStack Start**がベータ版のためメインオプションに含まれなかったが、自由回答で最多（235件）を獲得。来年の正式ランキング入りが確実視される

**ペインポイント：**

2. Next.js（241件）
3. SSR（110件）
4. パフォーマンス（100件）
5. 破壊的変更（91件）
6. デプロイメント（86件）
7. ドキュメント不足（83件）
8. Vercel（81件）
9. フロントエンド・バックエンド統合（73件）
10. ロックイン（70件）

### バックエンドフレームワーク

**ハイライト: ElysiaJS**
- **Bun**上に構築された唯一のバックエンドフレームワーク
- 新しいランタイムへの賭けがBunの人気上昇とともに報われつつある

**主要な発見：**
- **Express**が15年経った今でも使用率でリーダー
- **NestJS**が着実に成長中
- 満足度トップ3は**Hono、Nitro、ElysiaJS**とすべて新規参入者
- 平均使用バックエンドフレームワーク数は**1.9個**
- 型安全なTypeScript対応APIの提供が求められている

**ペインポイント：**
1. Express（80件）

3. ブラウザサポート（59件）

### テストツール

**ハイライト: bun test**
- テストランナーがランタイムに組み込まれるトレンドの一例

- テストツールの幸福度は5点満点中**3.5**（わずかに上昇傾向）

### ビルドツール

**ハイライト: Rolldown**
- ViteチームがRustベースのバンドラーとして開発中
- **Vite自体がRolldownを使用**するようになり、ビルドツールエコシステムの中心的存在に

**主要な発見：**

- **Turbopack**はNext.jsとのバンドルにより使用率が伸びているが、満足度は低い

## 重要な事実・データ

- **Vite（98%）とVitest（97%）** が満足度でトップ2を独占
- **Hono（95%）とBun（91%）** が今年新たにS-tierに参入
- **React**の満足度はB-tier（72%）に低下し、Vue.js（84%）やSvelte（86%）より下位
- **Next.js**は2025年に初めてネガティブ評価に転落（-3.08%）、C-tier入り
- テストツールの平均使用数**4.4個**は全カテゴリ中最多（決定版ツールがまだない）
- フロントエンドフレームワークの平均使用数は**2.6個**で、頻繁な乗り換えのイメージとは異なる
- **TanStack Start**が自由回答で最多の235件を獲得し、来年の正式参入が確実
- **Playwright**が2020年の-93%から2025年の+43%へと劇的な成長

## 結論・示唆

### 全体的な結論

1. **Viteエコシステムの支配**：Vite（98%）、Vitest（97%）、Rolldown（Viteの次世代バンドラー）がビルド・テストの領域を席巻。Viteを中心としたエコシステムが今後のJavaScript開発の標準になりつつある

2. **Reactの相対的地位低下**：使用率は依然高い（83.6%）が、満足度は72%でB-tier。ポジティブ評価は2016年の68.5%から2025年の31.0%に半減。ただし使用率自体は安定しており、「不満はあるが使い続けている」状態

3. **Next.jsの分極化**：使用率は高いが、「過度な複雑さ」「Vercelへのロックイン」「破壊的変更」が繰り返し指摘される。2025年に初めてネガティブ評価に転落

4. **新興ランタイムの台頭**：Bun（S-tier、91%）とそのエコシステム（ElysiaJS、bun test）が急成長。Denoも複数カテゴリで存在感を示す

5. **型安全への強い需要**：バックエンドでの「静的型付け」への不満、oRPCやtRPCの台頭が示すように、TypeScript対応の型安全なAPIが強く求められている

### 実践的な示唆

- **新規プロジェクト** → Vite + Vitest の組み合わせが最も安定した選択肢
- **E2Eテスト** → Playwright が最も高い満足度と成長率を示す
- **バックエンド** → Hono、Fastify が満足度の高い選択肢。Expressからの移行を検討する価値あり
- **メタフレームワーク** → Astro が満足度トップ。Next.jsを選ぶ場合はトレードオフを理解した上で
- **注目すべき新技術** → TanStack Start、Rolldown、Bun、oRPC

## 制限事項・注意点

- 調査結果は自己申告に基づくサーベイであり、実際の市場シェアとは異なる可能性がある
- 英語圏のJavaScript開発者に偏る傾向がある
- 「満足度」の低さは必ずしもツールの品質の低さを意味せず、期待値の高さや使用範囲の広さを反映している場合がある（例：React、Next.js）
- バックエンドフレームワークは3年ぶりの調査復帰であり、経時変化データが限定的

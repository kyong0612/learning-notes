## 概要

AIコーディングツール市場において、Claude Code が爆発的な成長でCursorを凌駕しつつある中、Cursorの「自社開発モデル」Composer 2 が実は中国 Moonshot AI のオープンソースモデル **Kimi K2.5** を基盤としていたことが発覚した事件を軸に、AI業界における米中の技術依存関係と「地政学的アービトラージ」の構造を論じる記事。タイトルの「ComposerをDecomposeする」は、Cursorの製品名であるComposer（作曲家）を分解（Decompose）すると中国の技術基盤が見えてくるという二重の意味。

## 主要なトピック

### Claude Code vs Cursor ― AIコーディングツールの成長競争

AIコーディングツールの二強であるClaude CodeとCursorの凄まじい成長曲線を比較。

- **Cursor（Anysphere社）**: 2023年デビュー → 2024年末 ARR $100M → 2025年7月 $500M → 同年11月 $1B → 2026年2月 $2B
- **Claude Code**: 2025年5月リリース → たった半年で ARR $1B突破 → 2026年3月 $3B ARR達成 → **Cursorを逆転**

AIコーディングエージェントの能力は一般人の知能を超えており、「ハーネス」を適切に設計すれば、月3万円足らずで「きちんと言うことを聞く優秀なオフィスワーカー」が雇えるようになった。Claude Codeのソースコードが流出して話題になったが、ソースコードをパクられたところで勢いは失わないだろうと著者は指摘。

### パクリをパクったらモウマンタイ ― Composer 2 と Kimi K2.5 事件

> ※ セクションタイトルは広東語の「冇問題（モウマンタイ）＝ノープロブレム」をかけた言葉遊び

2026年3月19日、Cursor は新モデル **Composer 2** を発表。公式ブログでは「継続的事前学習と強化学習の組み合わせ」による性能向上を強調し、基盤モデルの出所には一切言及しなかった。

#### 発覚の経緯

- リリースからわずか2時間後、開発者 @fynnso が API リクエストのデバッグ中にモデルIDを捕捉：`kimi-k2p5-rl-0317-s515-fast`
- これは文字通り **Kimi K2.5** + 強化学習（RL）であることを意味
- Moonshot AI の事前学習責任者が Tokenizer の一致を確認し、「ほぼ確実にKimiのモデルが後訓練された結果」と指摘
- テスラCEO マスクも「はい、KimiK2.5だ」とコメント

#### ライセンス問題

- Kimi K2.5 は**修正版MITライセンス**を採用
- 商業製品がMAU 1億超 or 月間売上$20M超の場合、UIに「Kimi K2.5」の表記義務
- Cursorの年間売上 $2B（月額約$167M）は明らかにこの閾値を超えていた
- にもかかわらず、Cursorは表記を行っていなかった

#### 前科：Composer 1 と DeepSeek

- **2025年11月**の Composer 1 リリース時にも、TokenizerがDeepSeekと一致することがコミュニティに発見されていた
- 推論中に時折**中国語を出力**するという現象も報告されていた
- 当時もCursorは説明を行わなかった
- つまり**中国のオープンソースモデルを非開示で使用するのは2度目**

#### 決着と反転

- **Moonshot AI（Kimi公式）** が声明を発表：CursorはFireworks AI経由の**認可済み商業パートナーシップ**でKimi K2.5を使用と確認、「祝賀」の姿勢に転換
- **Cursor共同創業者 Aman Sanger**: 「Kimi K2.5は評価した中で最強の基盤モデルだった。ブログ発表時にKimiに言及しなかったのは失敗」と謝罪
- **Lee Robinson（開発者教育VP）**: 最終モデルの計算量のうちベースモデル由来は**約1/4**、残り**3/4**はCursor独自のトレーニング

#### 地政学的アービトラージの構造

- 評価額 $293B の Anysphere が、評価額 $180B の月之暗面（Moonshot AI）の技術を「最強の基盤」として採用
- 米国のトップAIスタートアップが中国のオープンソースモデルに依存している構造が露呈
- Cursorが中国モデルの使用を隠蔽した動機は、急膨張する**バリュエーション**を支えるため
  - VS Code のフォークに過ぎないのに、「モデル研発能力を持つAI企業」としてのブランディングが必要だった
  - $50M（2023年）→ $400M → $2.6B → $29.3B → **$50B**（2026年3月交渉中）という急上昇を正当化する必要があった

## 重要な事実・データ

- **Cursor ARR推移**: $100M（2024末）→ $500M（2025.7）→ $1B（2025.11）→ $2B（2026.2）
- **Claude Code ARR推移**: リリース半年で$1B → $3B（2026.3）、Cursorを逆転
- **Anysphere評価額**: $50M → $400M → $2.6B → $29.3B → $50B交渉中
- **月之暗面（Moonshot AI）**: 評価額約$18B、3ヶ月で4倍以上に急伸、アリババ・テンセントが投資
- **Composer 2価格**: 入力$0.50/100万トークン、出力$2.50/100万トークン（前世代比で大幅値下げ）
- **AIコーディングエージェントのコスト**: 月額約3万円で「早慶〜旧帝大レベルのオフィスワーカー」相当
- **OpenRouter統計**: 中国オープンソースモデルの週間API呼び出し量が米国モデルを2週連続で上回る

## 結論・示唆

### 著者の結論

「ComposerをDecomposeすると見えてくる地政学的アービトラージ」とは、表面上は米国スタートアップの自社技術に見えるものが、実際には中国のオープンソースモデルを基盤とし、その技術力の差分を企業価値として取り込んでいる構造を指す。インターネット時代には「Copy to China」が常識だったが、AI時代には **「Copy from China」** が現実となっている。技術流動の方向が双方向・多方向に変化した。

### 実践的な示唆

- 中国発オープンソースモデル（Kimi K2.5, DeepSeek等）は世界のAI技術スタックを形作る最大の力になりつつある
- AI業界の前線は「誰がゼロからモデルを訓練するか」ではなく「誰が最速で適応・微調整・製品化するか」の競争に移行
- AIツール選定時には、基盤モデルの出所と地政学的リスクを考慮すべき
- バリュエーション競争がモデルの透明性開示を阻害するインセンティブ構造がある

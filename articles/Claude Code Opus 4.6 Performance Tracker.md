## 概要

[Margin Lab](https://marginlab.ai/) が提供する独立サードパーティ製のパフォーマンストラッカー。Claude Code CLI + Opus 4.6 の SWE タスク性能を毎日ベンチマークし、**統計的に有意な性能低下（デグラデーション）を検出する**ことを目的としている。

> **背景**: 2025年9月に Anthropic が公開した [Claude のデグラデーションに関するポストモーテム](https://www.anthropic.com/engineering/a-postmortem-of-three-recent-issues) を受け、将来的な性能低下を外部から検出するリソースとして構築された。

### 主な特徴

| 特徴 | 詳細 |
|------|------|
| **更新頻度** | 毎日 |
| **ベンチマーク対象** | [SWE-Bench-Pro](https://marginlab.ai/explorers/swe-bench-pro) の汚染耐性サブセット |
| **実行環境** | Claude Code CLI を直接使用（カスタムハーネスなし） |
| **使用モデル** | 最新の SOTA モデル（現在は Opus 4.6） |
| **独立性** | フロンティアモデル提供者との提携なし |

## 変化の概要（Change Overview）

各期間におけるベースラインからの差分。いずれも**統計的に有意ではない**と判定されている。

| 期間 | 差分 | 判定 | 有意性閾値 | テストケース数 |
|------|------|------|------------|----------------|
| **1日（昨日）** | -3.9% | Not Significant | ±13.5% (p < 0.05) | 50 |
| **7日（先週）** | -1.3% | Not Significant | ±5.2% (p < 0.05) | 300 |
| **30日（先月）** | +0.4% | Not Significant | ±2.3% (p < 0.05) | 1,450 |

**ポイント**: 日次では N=50 のため閾値が ±13.5% と広く、短期間の変動では有意差が出にくい設計。サンプル数が増える週次・月次では閾値が狭まり、より繊細な検出が可能になる。

## トレンド

### 日次トレンド（Daily Trend）

- パスレートの推移を時系列で表示
- 56% ベースラインの破線と ±13.5% 有意性閾値を併記
- 95% 信頼区間（CI）のトグル表示が可能

### 週次トレンド（Weekly Trend）

- 7日間集約のパスレートを表示
- ±5.2% 有意性閾値
- 同様に 95% CI のトグル表示対応

---

## その他の指標（Other Metrics）

日次ベンチマークのリソース消費と実行状況のトレンドとして、以下の5つのメトリクスがチャートで提供されている：

1. **Input Tokens** — 日次の入力トークン総使用量
2. **Output Tokens** — 日次の出力トークン総使用量
3. **API Cost** — 日次ベンチマークコスト（USD）
4. **Average Runtime** — インスタンスあたりの平均実行時間（日別）
5. **Tool Calls** — 日次のツール呼び出し総数

---

## 方法論（Methodology）

1. **テスト設計**: 各テストを**ベルヌーイ確率変数**としてモデル化
2. **日次評価**: N=50 テストインスタンスで実行（日次の変動は想定内）
3. **集約**: 週次・月次の結果を集約し、より信頼性の高い推定を提供
4. **統計検定**: 日次・週次・月次のパスレートに対して **95% 信頼区間**を算出
5. **検出報告**: いずれかの時間軸で統計的に有意な差異が検出された場合に報告
6. **環境**: Claude Code CLI を直接使用し、カスタムハーネスを用いないため、実ユーザーが期待できる結果を反映
7. **検出対象**: モデル変更とハーネス変更の両方に起因するデグラデーションを検出可能

---

## 重要な結論・知見

- **現時点でOpus 4.6の性能低下は検出されていない**（ステータス: Nominal）
- 30日間のパスレート（56%）はベースラインと一致しており、安定した性能を示している
- 日次 N=50 のサンプルサイズでは ±13.5% の変動が有意性閾値となるため、小さな変動は検出できない制約がある
- 週次（±5.2%）・月次（±2.3%）と集約期間が長くなるほど、より微細な変化を検出できる

## 制限事項

- **日次サンプルサイズが小さい**（N=50）：日ごとの変動が大きく、短期間の性能低下を見逃す可能性がある
- **SWE-Bench-Pro のサブセットのみ**を使用しており、すべてのソフトウェアエンジニアリングタスクを網羅しているわけではない
- グラフやチャートはインタラクティブ要素のため、ライブサイトでの確認が必要
- ベンチマーク結果は Claude Code CLI 経由の実行に限定されており、API 直接呼び出しの性能とは異なる可能性がある

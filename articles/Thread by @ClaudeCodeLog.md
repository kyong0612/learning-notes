# Claude Code 2.1.14 Changelog

**Claude Code Changelog** @ClaudeCodeLog [2026-01-21](https://x.com/ClaudeCodeLog/status/2013779031703851488)

> Claude Code 2.1.14 is out.
> 16 CLI, 5 flag, and 4 prompt changes.
> Details in thread ↓

---

## 概要

Claude Code 2.1.14は2026年1月20日にリリースされました。このバージョンでは、Bashモードの使い勝手向上、プラグイン管理の改善、および複数の重要なバグ修正が含まれています。

---

## 新機能

### 1. Bashモードの履歴ベース自動補完
- Bashモード（`!`）でコマンドの一部を入力し、**Tab**キーを押すとコマンド履歴から補完できるようになりました
- シェルヒストリーを活用してより素早いコマンド入力が可能に

### 2. インストール済みプラグインの検索機能
- プラグインリストで名前や説明でフィルタリングして検索可能に
- 多数のプラグインを管理しやすくなりました

### 3. プラグインのGitコミットSHA固定
- プラグインを特定のgitコミットSHAに固定できるようになりました
- マーケットプレイスのエントリから正確なバージョンをインストール可能
- 再現性のある環境構築に貢献

### 4. `/usage`コマンド（VSCode限定）
- 現在のプラン使用状況を表示するコマンドを追加

---

## バグ修正

### コンテキストウィンドウとパフォーマンス

#### コンテキストウィンドウのブロック制限の修正（重要）
- **問題**: コンテキストウィンドウのブロック制限が過度に厳しく計算され、本来の~98%ではなく~65%の使用率でユーザーがブロックされていた
- **修正**: 意図した~98%の閾値で正しく動作するように修正

#### 並列サブエージェント実行時のメモリクラッシュ修正
- 並列サブエージェント実行時にクラッシュを引き起こす可能性があったメモリ問題を修正

#### 長時間セッションのメモリリーク修正
- シェルコマンド完了後にストリームリソースが適切にクリーンアップされなかった問題を修正
- 長時間セッションの安定性が向上

### UI/UX改善

#### Bashモードでの`@`シンボル問題
- Bashモードで`@`シンボルがファイル自動補完を誤ってトリガーする問題を修正

#### `@`メンションメニューのフォルダクリック動作
- フォルダをクリックした際、選択ではなくディレクトリ内に移動するように修正

#### `/feedback`コマンドのURL生成
- 説明が長い場合に無効なGitHub Issue URLが生成される問題を修正

#### `/context`コマンドのトークン表示
- verboseモードでステータスラインと同じトークン数・パーセンテージを表示するように修正

#### コマンドオーバーレイの予期しない終了
- `/config`、`/context`、`/model`、`/todos`のオーバーレイが予期せず閉じる問題を修正

#### スラッシュコマンド自動補完
- 類似したコマンド（例: `/context` vs `/compact`）を入力時に間違ったコマンドが選択される問題を修正

### ナビゲーションとセッション管理

#### プラグインマーケットプレイスの戻るナビゲーション
- マーケットプレイスが1つだけ設定されている場合の戻るナビゲーションの不整合を修正

#### iTerm2プログレスバーのクリア
- 終了時にプログレスバーが適切にクリアされず、インジケーターやベル音が残る問題を修正

### 入力と編集

#### ペーストしたテキストのバックスペース動作
- ペーストしたテキストを削除する際、1文字ずつではなく単一トークンとして削除するように改善

## 概要

**RuView**は、WiFi信号を利用してカメラやウェアラブルデバイスなしに人体の姿勢推定、バイタルサイン監視、存在検知を行うエッジAI知覚システム。Carnegie Mellon大学の「DensePose From WiFi」研究を実用的なエッジシステムに拡張したプロジェクトで、[RuVector](https://github.com/ruvnet/ruvector/)をベースに構築されている。

**核心コンセプト**: WiFiルーターが発する電波が人体に当たると散乱パターンが変化する。RuViewはCSI（Channel State Information）の乱れを解析し、体位、呼吸数、心拍数、存在を物理ベースの信号処理と機械学習でリアルタイム再構成する。

## 主要機能

### センシング機能

| 機能 | 詳細 |
|------|------|
| **プライバシーファースト** | WiFi信号のみ使用。カメラ・映像・画像の保存なし |
| **バイタルサイン** | 呼吸数（6-30回/分）、心拍数（40-120 bpm）をウェアラブルなしで検出 |
| **複数人追跡** | 複数人を同時追跡、各人に独立した姿勢・バイタル情報（物理的に1 AP当たり3-5人、マルチAPで線形スケール） |
| **壁越しセンシング** | WiFiは壁・家具・瓦礫を透過。カメラが使えない場所でも機能。フレネルゾーンモデリングで最大5m深度 |
| **災害対応** | 瓦礫下の生存者検知、START トリアージによる重症度分類 |
| **マルチスタティックメッシュ** | 4-6ノードで12以上の重複信号経路を形成し、360度カバレッジ・サブインチ精度を実現 |
| **持続的フィールドモデル** | 部屋のRF特徴を学習し、環境を差し引いて人間の動きを分離。ドリフト検知・意図予測・スプーフィング検知も可能 |

### インテリジェンス機能

| 機能 | 詳細 |
|------|------|

## 動作原理

```
WiFiルーター → 電波が室内を通過 → 人体に衝突 → 散乱
    ↓
ESP32メッシュ（4-6ノード）がチャネル1/6/11でCSIをTDMプロトコルで取得
    ↓
マルチバンド融合: 3チャネル × 56サブキャリア = 168仮想サブキャリア/リンク
    ↓
マルチスタティック融合: N×(N-1)リンク → attention重み付きクロスビューポイント埋め込み
    ↓
コヒーレンスゲート: Accept/PredictOnly/Reject/Recalibrate → チューニング不要で安定
    ↓
信号処理: Hampel, SpotFi, Fresnel, BVP, スペクトログラム → クリーンな特徴量
    ↓
AIバックボーン（RuVector）: attention, グラフアルゴリズム, 圧縮, フィールドモデル
    ↓
ニューラルネットワーク: 処理済み信号 → 17体キーポイント + バイタルサイン + 部屋モデル
    ↓
出力: リアルタイム姿勢, 呼吸, 心拍, 部屋フィンガープリント, ドリフトアラート
```

## ハードウェア要件

> **重要**: 姿勢推定・バイタルサイン・壁越しセンシングにはCSI対応ハードウェアが必要。一般的な消費者向けWiFiではRSSIベースの存在検知のみ。

| オプション | ハードウェア | コスト | 完全CSI | 対応機能 |
|------------|-------------|--------|---------|----------|
| **ESP32メッシュ（推奨）** | 3-6x ESP32-S3 + WiFiルーター | ~$54 | Yes | 姿勢・呼吸・心拍・動作・存在 |
| **研究用NIC** | Intel 5300 / Atheros AR9580 | ~$50-100 | Yes | 3x3 MIMO完全CSI |
| **任意のWiFi** | Windows, macOS, Linux | $0 | No | RSSIのみ: 粗い存在・動作検知 |

## 自己学習WiFi AI（ADR-024）

WiFi信号を128次元の「フィンガープリント」ベクトルに変換し、部屋認識・侵入検知・個人再識別・活動分類を実現する自己教師あり学習システム。

| 機能 | 詳細 |
|------|------|
| **自己教師あり学習** | ラベルなしのWiFiデータだけで訓練可能。10分プラグインするだけで動作 |
| **部屋識別** | 各部屋が固有のWiFiフィンガープリントパターンを生成 |
| **異常検知** | 未知の人物やイベントが既存パターンと不一致で自動検出 |
| **個人再識別** | 個人ごとのWiFi攪乱パターンでカメラなしにトラッキング |
| **環境適応** | MicroLoRAアダプター（部屋あたり1,792パラメータ）で少量データで適応 |

**モデルサイズ**: 合計約55,000パラメータ / 55 KB（ESP32の利用可能520KBのうち）

## クロス環境汎化（ADR-027 — Project MERIDIAN）

1回の訓練でどの部屋にも再訓練なしにデプロイ可能にするシステム。

| 手法 | 機能 |
|------|------|
| **Gradient Reversal Layer** | 敵対的分類器が部屋を推測→ネットワークは部屋固有のショートカットを破棄 |
| **Geometry Encoder（FiLM）** | 送受信位置をフーリエエンコード→各層にスケール+シフト条件付け |
| **Hardware Normalizer** | あらゆるチップセットのCSIを標準56サブキャリア形式にリサンプル |
| **Virtual Domain Augmentation** | 2-3部屋のデータから数千の合成環境を生成 |
| **Rapid Adaptation（TTT）** | ゼロショットデプロイ：数フレームで自己チューニング |

## RVFモデルコンテナ

訓練済みモデルを単一バイナリファイル（`.rvf`）にパッケージング。重み、HNSWインデックス、量子化コードブック、SONAアダプタ、WASM推論ランタイムを含む。

| デプロイ先 | 量子化 | サイズ | ロード時間 |
|-----------|--------|--------|-----------|
| ESP32/IoT | int4 | ~0.7 MB | <5ms |
| モバイル/WebView | int8 | ~6 MB | ~200ms |
| ブラウザ（WASM） | int8 | ~10 MB | ~500ms |
| フィールド（WiFi-Mat） | fp16 | ~62 MB | ~2s |
| サーバー/クラウド | f32 | ~50+ MB | ~3s |

**プログレッシブローディング**: Layer A（即時）→ Layer B（ウォーム）→ Layer C（フル）の3段階ロード。

## 制限事項

- **CSI対応ハードウェアが必要**: 完全な姿勢推定・バイタルサインには ESP32-S3 や研究用NICが必要。一般消費者向けWiFiではRSSIベースの粗い存在検知のみ
- **物理的制約**: 1つのAPあたり56サブキャリアで約3-5人まで。それ以上はマルチAPメッシュが必要
- **壁越し性能**: 非金属材料（コンクリート・木・ドライウォール）は透過するが、金属構造では性能低下
- **ベンチマーク検証**: 監査時点でベンチマークスループットとKubernetesデプロイは未計測（33項目中2項目）
- **ESP32ファームウェアv0.3.0**: まだアルファ段階（エッジインテリジェンス機能）

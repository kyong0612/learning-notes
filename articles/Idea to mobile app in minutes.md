## 概要

**Rork**は、自然言語の説明からiOS/Androidのネイティブモバイルアプリを自動生成するAI搭載ノーコードプラットフォーム。Levan KvirkveliaとDaniel Dhawanが創業し、a16z（Andreessen Horowitz）から**280万ドル**の資金調達を実施。月間**74万3,000以上**のアクセスを集め、**85%**の成長率を記録している。

## Rork（標準版）

- **技術スタック**: React Native + Expo
- **対応プラットフォーム**: iOS、Android、Web（クロスプラットフォーム）
- 自然言語プロンプトから機能的なアプリコードを自動生成
- ユーザー認証、データストレージ、ナビゲーションなどを自動組み込み
- Expo Go経由でQRコードスキャンによるリアルタイムプレビュー
- EAS（Expo Application Services）によるApp Store/Google Play直接公開
- GitHub連携によるコードエクスポート・バージョン管理
- 生成コードの完全な所有権をユーザーが保持

## Rork Max（上位版）

Rork Maxは**Swift/SwiftUI**ベースのネイティブiOSアプリビルダーで、ブラウザ上からネイティブiOSアプリを構築できる世界初のプラットフォーム。

### 対応Appleプラットフォーム

iPhone、iPad、Apple Watch、Apple TV、Vision Pro、iMessage

### 高度な機能

| カテゴリ | 対応機能 |
|---|---|
| UI/UX | ウィジェット（ホーム画面/ロック画面）、Dynamic Island、Live Activities、スプリングアニメーション |
| ゲーム | 3Dゲーム（SceneKit/Metal）、物理演算、ARゲーム |
| AR/カメラ | ARKit、LiDAR、ボディトラッキング、顔検出、Snapchat風レンズ |
| システム連携 | Siriインテント、HealthKit、HomeKit、NFC、App Clips、カスタムキーボード |
| 通信 | iMessageアプリ、Multipeer（ローカルマルチプレイヤー）、バックグラウンドタスク |

### 技術的仕組み

- クラウド上のMac群で24時間稼働するXcode + iOS SDK環境
- AIエージェントがSwiftUIでコードを書き、ビルド→エラー修正→再ビルドのサイクルを自動化
- FaceTimeと同じ低遅延プロトコルで実際のiOSシミュレーターをブラウザにストリーミング
- ワンクリックでiPhoneへインストール、ツークリックでApp Storeへ提出

## 利用フロー

1. **アイデアを記述** — 自然言語でアプリの概要・機能を説明
2. **AI が生成** — 画面構成、ナビゲーション、コア機能を自動構築
3. **チャットで改善** — 会話形式でデザインや機能を反復的に調整
4. **ライブテスト** — Expo GoのQRコード or ブラウザ内シミュレーターで即座に確認
5. **公開** — App Store / Google Play / TestFlightへ直接提出

## 強み

- **開発速度**: 数週間かかるMVPを数時間で構築可能
- **コーディング不要**: 完全にノーコードでアプリ作成
- **ネイティブ体験**: Webラッパーではない本物のネイティブアプリ
- **クロスプラットフォーム**: 単一コードベースで全プラットフォーム対応
- **コード所有権**: React Nativeソースコードをいつでもエクスポート可能
- **Rork Max独自**: ブラウザからSwiftUIアプリを構築できる唯一のプラットフォーム

## 制限事項

- **無料トライアルなし** — 試用前に有料プランへのコミットが必要
- **安定性の課題** — バグやクラッシュ、データ損失の報告あり
- **カスタマイズの限界** — ビジュアルエディタ未搭載、細かいデザイン制御にはコーディング知識が必要
- **カスタマーサポート** — レスポンス時間が遅い、ドキュメント不足
- **メッセージ制限** — 複雑なプロジェクトでは反復によりメッセージを急速に消費
- **Trustpilot評価**: 2.9/5（レビュー数は限定的）

## 最適なユーザー層

- コーディングスキルのない起業家（アイデア検証、MVP構築）
- デザイナー（静的モックアップからインタラクティブなプロトタイプへ）
- 小規模ビジネス（低予算でモバイルプレゼンスを確立）
- 学生（アプリ開発の学習、ポートフォリオ構築）
- エージェンシー（クライアントへの動作デモ提示）

エンタープライズ規模の複雑なアプリ、ピクセル単位の精密なカスタマイズ、堅牢なサポートが必要な場合は他のソリューションを検討すべき。

# ソフトウェアは捨てやすく作ろう

ref: <https://qiita.com/sanogemaru/items/40557c6db33dcec99cf1>

- **ソフトウェア開発における高い不確実性**: ソフトウェア開発では、提案された製品が実際の問題を解決できるかどうか、そして収益性があるかどうかについて、高い不確実性に直面することが多い。
- **継続的な改善の必要性**: 迅速な仮説検証を通じて製品を絶え間なく改善していくという姿勢は、効果的なソフトウェア開発にとって不可欠である。
- **削除可能性の重要性**: 開発プロセス中、特にプロダクト市場適合性(PMF)を達成する前に、不要な機能が生じることがある。そのため、簡単に破棄できるソフトウェアを構築することが重要である。
- **未使用コードによる認知負荷**: 不要なコードが蓄積されると、開発者の認知負荷が増加し、コード品質の低下や、ビジネス成果への悪影響につながる可能性がある。
- **チームの連携**: 簡単に破棄できるソフトウェアを作るという原則に開発チームが一致することは重要であり、「削除する勇気を持つ」文化を醸成することが不可欠である。
- **管理しやすいディレクトリ構造**: レイヤーではなく機能に基づいたディレクトリ構造を採用することで、コードの管理性が向上し、特定の機能を簡単に削除できるようになる。
- **高凝集性と低結合性**: コードデザインにおいて高い凝集性と低い結合性を確保することで、機能の管理と削除が容易になる。
- **管理の拡張性**: ソフトウェアが拡大するにつれ、多数の機能に対応できる管理可能なディレクトリ構造を維持することが、効果的な仮説検証のために重要となる。
- **コード品質のビジネスへの影響**: コードの品質はビジネスパフォーマンスに直接影響するため、コードの保守性と明確性を優先することが不可欠である。
- **ベストプラクティスの採用**: 「機能別パッケージ化」などのプラクティスを実践することで、変化する要件に迅速に適応し、ソフトウェアライフサイクルを効果的に管理することができる。

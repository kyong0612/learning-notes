篠塚氏が令和トラベルで取り組んでいるプロダクト・プロジェクトの紹介セクション。同氏は旅行アプリ「NEWT（ニュート）」を運営し、2025年7月にはAIチャットエージェント「NEWT Chat」を発表。旅行・サービス業界に特化したAIエージェント事業を展開している。

### Agent Teams で何が変わったか

Claude Code Agent Teamsは、**複数のClaude Codeインスタンスをチームとして協調動作させる機能**で、Claude Opus 4.6と同時にリサーチプレビューとしてリリースされた。記事では以下の3つの変化点が取り上げられている。

#### 1. チームメイト同士が直接会話できる

Agent Teamsでは、各エージェント（チームメイト）が**独立したコンテキストウィンドウ**を持ちながら、**メッセージングシステムを通じて直接コミュニケーション**できる。従来のサブエージェントでは結果をメインエージェントに返すだけだったが、Agent Teamsでは以下が可能になった：

- **メッセージ（message）**: 特定のチームメイトに直接メッセージを送信
- **ブロードキャスト（broadcast）**: 全チームメイトに同時送信
- **共有タスクリスト**: 全エージェントがタスク状態を確認し、利用可能な作業を自ら取得

これにより、例えばセキュリティ担当、パフォーマンス担当、テストカバレッジ担当のエージェントがそれぞれの知見を共有し、互いの発見に挑戦し合うような協調作業が可能になった。

#### 2. 同じメンバーに「修正して」が言える

Agent Teamsでは各チームメイトが**永続的なセッション**として動作するため、同じエージェントに対して追加の指示や修正依頼を直接送ることができる。

- **In-processモード**: Shift+Up/Downでチームメイトを選択し、直接メッセージを入力
- **Split-paneモード**: チームメイトのペインをクリックして直接対話

従来のサブエージェントは一度結果を返したら終了だったが、Agent Teamsのメンバーは継続的に作業を進め、フィードバックを受けて修正できる。

#### 3. 自律的に仕事を取りに行く

Agent Teamsの重要な特徴として、チームメイトが**タスクを自ら取得（self-claim）**する機能がある。

- リードエージェントがタスクを明示的に割り当てることも可能
- チームメイトが現在のタスクを完了すると、**次の未割り当て・ブロック解除済みのタスクを自動的に取得**
- タスクの競合を防ぐファイルロック機能を搭載
- タスク間の依存関係も自動管理（依存先が完了すると、ブロックされていたタスクが自動解除）

### 実際に動かしてみた

篠塚氏がAgent Teamsを実際のNEWT開発環境で試した実践報告。Agent Teamsが最も効果を発揮するユースケースとして、公式ドキュメントでは以下が挙げられている：

- **リサーチ&レビュー**: 複数のチームメイトが問題の異なる側面を同時に調査
- **新モジュール・機能開発**: 各チームメイトが異なるモジュールを担当
- **競合仮説によるデバッグ**: 異なる理論を並行でテストし、収束を加速
- **クロスレイヤー連携**: フロントエンド、バックエンド、テストをそれぞれ担当

### 学びと気づき

Agent Teamsを使った開発から得られた知見。公式ドキュメントのベストプラクティスと照らし合わせると、以下のような学びが考えられる：

- **十分なコンテキストの提供**: チームメイトはリードの会話履歴を引き継がないため、スポーンプロンプトにタスク固有の詳細を含める必要がある
- **適切なタスクサイズ**: 小さすぎると協調オーバーヘッドが価値を上回り、大きすぎるとチェックインなしに長時間作業して無駄が増える
- **ファイル競合の回避**: 2つのチームメイトが同じファイルを編集すると上書きが発生するため、各チームメイトが異なるファイルセットを担当する設計が重要
- **トークン使用量**: Agent Teamsは単一セッションよりも大幅にトークンを消費する

## 背景情報

### Claude Opus 4.6について（2026年2月5日リリース）

- **1Mトークンのコンテキストウィンドウ**: Opusクラスモデル初
- **Agent Teams**: Claude Codeでの研究プレビュー機能
- **Adaptive Thinking**: 深い推論が必要かをモデルが自律判断
- **Effort制御**: low/medium/high/maxの4段階
- **Context Compaction（β）**: 古いコンテキストを自動要約して長時間タスクを可能に
- **128kトークン出力**: 大規模な出力を1リクエストで完了
- **ベンチマーク**: Terminal-Bench 2.0で最高スコア、Humanity's Last Examで全フロンティアモデルをリード

### Agent Teams vs サブエージェント

| | サブエージェント | Agent Teams |
|---|---|---|
| **コンテキスト** | 結果をメインに返す | 完全に独立 |
| **コミュニケーション** | メインエージェントへの報告のみ | チームメイト同士が直接メッセージ |
| **協調** | メインエージェントが全作業を管理 | 共有タスクリストで自己協調 |
| **最適用途** | 結果のみ重要な集中タスク | 議論と協働が必要な複雑な作業 |
| **トークンコスト** | 低い | 高い（各チームメイトが個別インスタンス） |

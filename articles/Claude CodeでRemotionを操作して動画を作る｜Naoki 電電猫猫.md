## 概要

「動画を作りたいけど、After Effectsは難しい」「プログラミングで動画を作れるらしいけど、コードを書くのは面倒」という課題に対して、Claude CodeとRemotionを組み合わせることで、**自然言語で指示するだけで動画が作れる**という解決策を提案する記事。

## Remotionとは

Remotionは**React/TypeScriptでコードベースの動画制作を可能にするフレームワーク**。

### 従来の動画編集ソフトとの違い
- マウスでタイムラインを操作するのではなく、コードで動画を組み立てる
- フレーム単位での正確な制御が可能
- テキストの表示タイミングや音楽へのシンクが得意
- アニメーションの微調整が容易

### 基本的な仕組み
- **コンポジション（Composition）**: 動画の単位。フレーム数、FPS、解像度などを定義
- 動画の内容は通常のReact関数コンポーネントとして実装
- `useCurrentFrame`フックで現在のフレーム番号を取得し、アニメーションを制御

## Claude Codeとの連携

### なぜこの組み合わせが上手く機能するのか

1. **Remotionはコードベース**: Reactコンポーネントとして動画を構築するため、Claude Codeが直接コードを生成・編集できる
2. **即時プレビュー**: `npm run dev`でブラウザ上でリアルタイムプレビューが可能
3. **自然言語→コード変換**: Claude Codeが自然言語の指示を理解し、適切なTypeScriptコードに変換

### 制作ワークフロー

1. **セットアップ**
   - Node.js v18以上をインストール
   - `npx create-video@latest` でプロジェクト生成

2. **開発サイクル**
   - `npm run dev` でブラウザプレビュー開始
   - Claude Codeに自然言語で指示
   - `src/Video.tsx`にコードが自動生成される
   - プレビューで確認・調整

3. **書き出し**
   - `npx remotion render` でMP4ファイルに出力

## 効果的な指示の出し方

### ポイント
- 具体的なビジュアルイメージを言葉で伝える
- アニメーションのタイミングやイージングを指定
- 参考になる既存の動画やスタイルがあれば言及

### 指示例
「この音源と歌詞ファイルを読み込んでリリックモーションにして」といった自然言語での指示が可能。

## 実践例：ターミナル風アニメーション

記事では、実際にターミナル風のアニメーションを作成した過程を解説。動画自体もClaude Codeへの指示によって制作された。

## ターミナル環境の最適化

Claude Codeを快適に使うためのターミナル設定も重要。

### 推奨ターミナル
- **安定性重視**: iTerm2（Mac）/ Windows Terminal
- **高速処理**: Ghostty / WezTerm

### 設定のポイント
- `Shift+Enter`や`Option+Enter`を改行キーとして設定
- 複数行入力に対応させることで、より詳細な指示が可能に

## まとめ

Claude CodeとRemotionの組み合わせにより：
- **非エンジニアでも**自然言語で動画制作が可能
- **フレーム単位の正確な制御**とAIによる自動コード生成の両立
- プレビュー→修正→レンダリングの効率的なワークフローを実現

この手法は、プログラミングスキルがなくても高品質なアニメーション動画を制作したい人にとって、新しい選択肢となる。

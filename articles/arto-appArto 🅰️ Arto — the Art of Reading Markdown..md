## 概要

**Arto**（"Art of Reading" の略）は、Markdownを**美しく読む**ことに特化したmacOS向けローカルアプリケーション。多くのMarkdownツールが「書く」ことに焦点を当てる中、Artoは「読む」体験を重視し、GitHubスタイルのMarkdownレンダリングをオフライン環境で忠実に再現する。

- **リポジトリ**: [arto-app/Arto](https://github.com/arto-app/Arto)
- **公式サイト**: [arto-app.github.io](https://arto-app.github.io)
- **言語**: Rust
- **ライセンス**: MIT
- **スター数**: 103 / フォーク数: 7（2026年2月時点）

> [!WARNING]
> 現在ベータ版であり、バグや不安定な動作が含まれる可能性がある。機能は後方互換性を考慮せず変更される場合がある。**macOS専用**（クロスプラットフォーム対応は長期目標）。

## フィロソフィー

Markdownは軽量マークアップ言語を超え、ドキュメント・コミュニケーション・思考の媒体となった。Artoは「Markdownを読むことはユーティリティタスクではなく、静かで意図的な理解と鑑賞の行為」という哲学に基づき設計されている。コンテンツが主役となり、インターフェースが背景に退く静穏な空間を提供する。

## 主要機能

### コアリーディング体験

| 機能 | 説明 |
|------|------|
| **GitHub スタイルレンダリング** | 拡張構文を含むGitHubのMarkdownスタイリングを正確に再現 |
| **ネイティブパフォーマンス** | Rust製による高速でレスポンシブなレンダリング |
| **自動リロード** | ディスク上のファイル変更を検知して自動更新 |
| **オフラインファースト** | インターネット接続不要でいつでもどこでも閲覧可能 |

### ナビゲーション・整理

- **ファイルエクスプローラー**: ローカルディレクトリをブラウズするサイドバー付きファイルツリー
- **クイックアクセス**: よく使うファイル・ディレクトリのブックマーク
- **ディレクトリ履歴**: サイドバー内の戻る/進むナビゲーション
- **目次（TOC）パネル**: ドキュメント内の自動目次生成
- **ライブナビゲーション**: リンクされたMarkdownドキュメント間の履歴付き遷移

### 検索・発見

- **ページ内検索**: `Cmd+F` でドキュメント内検索
- **ピン留め検索**: 検索クエリをピン留めし、セッションをまたいで複数色のハイライトを永続化

### ウィンドウ・タブ管理

- **タブサポート**: 単一ウィンドウ内で複数ドキュメントをタブ管理
- **マルチウィンドウ**: 複数ウィンドウの作成、ダイアグラム用の子ウィンドウ
- **クロスウィンドウタブ**: ウィンドウ間でのタブのドラッグ＆ドロップ
- **ドラッグ＆ドロップ**: Markdownファイルをウィンドウにドラッグして開く

### 高度なレンダリング

| 機能 | 説明 |
|------|------|
| **Mermaidダイアグラム** | ズーム、パン、画像コピー機能付きインタラクティブビューア |
| **数式表示** | KaTeXによる美しい数式レンダリング |
| **コードハイライト** | シンタックスハイライト＋コピーボタン付きコードブロック |
| **Frontmatter** | YAMLフロントマターを折りたたみ可能なテーブルとして表示 |
| **GitHub Alerts** | NOTE, TIP, IMPORTANT, WARNING, CAUTION アラートの完全サポート |

### カスタマイズ

- **ダークモード**: システム設定に基づく手動/自動テーマ切替
- **ズームコントロール**: キーボードショートカットとトラックパッドジェスチャー
- **設定**: サイドバー、TOCなどの設定可能項目
- **コンテキストメニュー**: ファイルやコンテンツへの右クリックメニュー

## インストール方法

### Homebrew（推奨）

```bash
brew install --cask arto-app/tap/arto
xattr -dr com.apple.quarantine /Applications/Arto.app
```

> Apple Developer IDで署名・公証されていないため、quarantine属性の解除が必要。

### Nix

一時的に試す場合:

```bash
nix run github:arto-app/Arto
```

永続インストール（nix-darwin / home-manager）の場合は、flake inputsに追加:

```nix
arto.url = "github:arto-app/Arto";
```

## 使い方

```bash
arto                     # 起動（ウェルカム画面表示）
arto README.md           # 特定ファイルを開く
arto docs/               # ディレクトリをファイルエクスプローラーで開く
arto file1.md file2.md   # 複数ファイルをタブで開く
```

Artoはシングルインスタンスで動作し、既に起動中の場合はパスを既存プロセスに送信する。

## 重要なポイント

1. **「読む」に特化した唯一性**: 編集機能を持たず、Markdownの閲覧体験に完全に集中している
2. **GitHub互換性**: GitHubのMarkdownレンダリングを忠実に再現するため、GitHub上のドキュメントをローカルで同じ見た目で確認可能
3. **オフラインファースト**: ドキュメントはローカルに保持され、プライバシーとアクセシビリティを確保
4. **ベータ段階**: 現時点ではmacOS専用で、クロスプラットフォーム対応はPR歓迎の長期目標

## 人生の成功に欠かせない最重要能力「実行機能」を整えよう！#7「メタ学習ループ」

『「実行機能」を整えよう！』の続きです！([#1](https://ch.nicovideo.jp/paleo/blomaga/ar2224861),[#2](https://ch.nicovideo.jp/paleo/blomaga/ar2224927),[#3](https://ch.nicovideo.jp/paleo/blomaga/ar2224995),[#4](https://ch.nicovideo.jp/paleo/blomaga/ar2225716),[#5](https://ch.nicovideo.jp/paleo/blomaga/ar2225882),[#6](https://ch.nicovideo.jp/paleo/blomaga/ar2225996))

このシリーズでは、私たちの人生においてめっちゃ重要な「 **実行機能** 」をうまく働かせる方法をチェックしております。こいつは脳の“司令塔”のような存在で、実行機能が弱っていると、人生のあらゆる面がうまくいかなくなっちゃうんで、おおまかな仕組みを知っておくのはめっちゃ大事なんですよ。

で、前回でワーキングメモリの機能をサポートする方法を終えまして、今回はシリーズの最終回。ここまで述べてきたことを日々の実践に落とし込むために最も重要な「 ****方略**** 」の話をしましょう。

## “方略”ってなんぞや？

あらためて簡単にまとめると、このシリーズでは、以下のことを学んでまいりました。

- 私たちが抱える困難の多くは、性格ではなく実行機能の問題である。
- 実行機能の土台になるのは、注意・メタ認知・ワーキングメモリの3つである。
- 実行機能は心の余裕に大きく左右されるため、環境調整が最も重要になる。
- 実行機能の改善は「整える」が中心で、「鍛える」ではない。

これらの要素を満たすために、ここまであらゆる「実行機能のサポート法」をチェックしてきたわけですな。要するに、

- 毎日やることに追われて、ずっと何かに追い立てられてる感じがする
- 仕事に向かっているはずが、気がつけばYouTubeやSNSを巡回している
- 頭がパンパンなのに、なぜか集中できない

みたいな状態は、脳科学でいうと「実行機能が削られている状態」とも言えまして、これをリカバリするために、環境を整えてやる必要があるわけっすね。

が、当たり前ながら、たんに知識を知っているだけでは行動は変わらないし、パフォーマンスも上がらないわけです。頭で分かっている知識を、日々の行動に“染み込ませる”ためには、「自分なりのやり方」を意識的に扱えるようになる必要がありますからね。

そこで最後に必要となるのが「方略」でして、これは「目標達成のために、自分が選んで使う“やり方”」を意味します。たとえば、

- プレゼンをつくるとき、まず構成をメモに書いてからスライドに入る → これは“構成重視”の方略だと言える
- 本を読むとき、先に目次をチェックしてから読む → これは“見通し型”の方略だと言える
- 仕事で詰まったら、紙に頭の中を書き出して整理する → これは“外在化”の方略だと言える
- 仕事のタスク整理で、重要度順に並べて優先順位をつける → これは“優先化”の方略だと言える
- 新しいスキルを習得する時に、手を動かす前に成功例を集めて真似する → これは“モデリング”の方略だと言える
- 会議の準備で、相手の関心を想定して逆算する → これは“逆算型”の方略だと言える
- 複雑な資料作成をする際に、一度紙に手書きで構成を書き出す → これは“視覚化”の方略だと言える

みたいなことを意味してるんですね。要するに、自分が「やってること」ではなく、「 **どうやってやってるか** 」に着目するのが「方略」であります。

なぜ“方略”が大事なのかと言いますと、これが実行機能と深くつながる要素だからです。これは非常に簡単な話で、

- メタ認知やワーキングメモリが弱い人ほど、「なんとなく」手を動かしがちで「どうやってやってるか」がわからない（つまり、方略を自覚できていない）
- メタ認知やワーキングメモリが使える人は、「この順番でやれば効率的」と、頭の中で戦略を選べる（つまり、方略がわかっている）

といったように、方略を持っていないか、自分の方略に気づいていない人ってのは、メタ認知とワーキングメモリがうまく働かず、「何から手をつけたらいいか分からない！」って状態にハマりがちなんですよ。

そのため、「方略の自覚」がないと、以下のような問題が起きるケースが多めであります。

- なんとなくやって → なんとなく失敗し → なぜ失敗したかもわからない
- 結果、再現性がないまま、次も同じミスを繰り返す
- 「自分には向いてない」「才能がない」と結論づけてしまう

いずれも落ち込んじゃうようなな問題ですけども、これはあなたに才能がないのではなく、“方略に気づいていない”ことが問題になってるだけなんで。

実際のところ、脳科学的に見ると、「学びの成果」を左右するのは才能でも根性でもなく、

1. どんな“方略”で学ぼうとしているか
2. それにちゃんと気づいているか
3. 使ったあとに振り返っているか

の3点に集約されると考えられるんですな。多くの人は“方略”を無意識に使ってるんだけど、それを **自覚して、振り返って、改善していく** ことで、爆発的に成果が上がるものなんですよ。

これは俗に“メタ学習ループ”とも呼ばれる方法でして、ここでやることは非常にシンプルです。

1. **自分が使っている方略に“気づく”** （例：「自分はいつも先延ばししてから焦ってやるな」）
2. **その方略を“実行してみる”** （今回は、逆に午前中に先にやってみる）
3. **“振り返る”** （前の方法と比べてどうだったか？）

この3ステップのループを回すことで、あなたは“方略”を意識的に使うことができるようになり、その結果として実行機能もブーストするわけです。

“メタ学習ループ”を理解するために、たとえば「プレゼン用の資料作成」を例にしてみると、

- **一般的なやり方：**
	- 先輩の資料をマネる
	- 手を動かして完成させる
	- フィードバックをもらうけど、改善は場当たり的
- **メタ学習型ループのやり方：**
	- **1．自分が使っている“資料作成の戦略”を言語化する** ：たとえば、まず全体構成を決めてから、1スライドずつ作る。参考資料はNotionにまとめる、など。
	- **2．実際にその戦略で取り組む** ：実行中に「これは順番間違えたな」と気づくのも大事な気づきになる。
	- **3．終わった後に振り返る** ：「参考資料が多すぎて迷った」「構成を先に決めたことでスムーズだった」などをメモする。

みたいになります。めっちゃ簡単ですけども、この“自己観察とフィードバック”のループを毎回まわすことで、「自分だけの攻略法」が少しずつ洗練されていくんですな。

## メタ学習ループを実践してみよう！

ってことで、「自分のやり方＝方略」に気づくために、メタ学習ループを実践してみましょう。自分の行動パターンに意識的になり、それを改善サイクルに乗せていくための具体的なステップは、以下のようになります。

## ステップ０　テーマを決める（対象を絞る）

まずは「何について上手くなりたいのか？」を明確にしましょう。たとえば、

- 企画書を効率よく作りたい
- 毎日のタスク処理をミスなく進めたい
- 資料読み込みの理解度を上げたい

みたいになります。“成果が出ていない”か、または“伸び悩んでいる”行動から選ぶのが一般的っすね。

## ステップ1　方略を自覚する

上にも書いたとおり、人は無意識に「自分なりの戦略（＝方略）」を使って行動しております。たとえば、資料を作るときに、「先に画像を探す人」や「文章から書く人」にわかれたりするじゃないですか。これは、みんな無意識のうちに「なんとなく自分に向いた方法」を選んでいるからです。

が、それに気づいていないと、自分のやり方のどこが強みで、どこが課題なのかも理解もできなくなり、結果として「どこを変えたら成果が出るのか」が分からず、“やみくもな努力”を続ける羽目になっちゃうんですよ。これはどうにかしておきたいところです。

方略に気づくためには、以下の手順をお使いください。

### 1-1．自分の「思考と行動の手順」をさかのぼる

まずは、以下のような質問で、自分がいつもどんなふうに作業をしているのかをチェックしてみましょう。

- 私はいつも、 **どこから手をつけているだろうか？** （例：リサーチ？ブレスト？スライドを開く？）
- 私はいつも、何を判断の軸にしているだろうか？（例：「手が早い方法」「前にうまくいった方法」「なんとなく安心できる順」）
- 私が自然に選んでいるやり方は、 **誰かの影響を受けていないだろうか？** （例：前職の上司、あの本、YouTube動画…）

たとえば、「営業の資料をつくる時に自分はどんな手順をふんでるのかなー？」とか考えた上で、

- まず過去資料を開く（安心感を得る）
- 顧客ニーズよりも、ストーリー構成から考える
- 頭で構成を作って、PowerPointを開く
- 途中で詰まると、別案件の資料に逃げがち

みたいなパターンがあることを自覚していくわけですね。こうして、「無意識だった行動のパターン」を、方略として捉え直していくわけです。

### 1-2．「いつも」と「たまに」の違いを比べてみる

続いて、以下のような質問で、自分に向いている“方略”をチェックしてみてください。

- うまくいったときと、うまくいかなかったとき、 **やり方の違いはなにがあった？**
- モチベーションが高い日、低い日で **手順や順番は変わっていた？**
- 成果が出た日は、 **「どこに集中」していた？**

たとえば、こんな感じで実践するわけです。

- **成果が出た日：**
	- 資料作成の前に「手書き」で構成を描いていた
	- 参考資料を“見ないで”自分で言語化した → 方略：「手を動かす前に“頭の中を出す”ことで集中が高まる」
- **ダメだった日** ：
	- 過去資料を開いたまま模倣に走って混乱 → 無意識の方略：「手癖で流しながら安心を得ようとする」

これによって、「どんな方略が自分にとって効果的なのか？」の仮説が立てられるわけですね。ちなみに、「方略の棚卸し」をする際には、ChatGPTに「最近、仕事で成果が出たプロジェクトで、自分が無意識に使っていた“やり方”を一緒に棚卸ししてください。」などと持ちかけてみると、自分では気づけない視点が引き出せたりしますんで、こちらもお試しください。これらの作業をやっとくと、自分がいつもやってたことの再現性が高まるし、不調なときにもその原因がわかるし、「何を変えればいいのか？」が見えやすくなりますんで。

## ステップ2　方略を意図的に使ってみる

このステップでは、「いつもの自分」を“あえて再現する”作業、または「いつもと違う方法」を“実験として試す”作業のいずれかを行います。

このステップを行う理由は、多くの人は以下のような落とし穴にハマりがちだからです。

- 「やり方を変えたい」と思っても、実行中に元のクセに戻る
- 「何が悪かったのか」振り返るときに、 **実際に“何をやったか”を覚えていない**
- 成果の出なさを「やる気」や「能力」のせいにしてしまう

どれもめっちゃよくある問題でして、こいつを解決するために、このステップでは、

- 行動を“仮説検証”の場にする
- “方略”を一時的に定数として固定し、「戦略を試す」意識で実行する
- 「感情」ではなく「プロセス」に焦点を当てる

みたいなことを行うわけです。つまり、このステップを行う時に必要なマインドってのは、

「これが正解かどうかはわからないけど、 **試してみて、手応えを確かめる** 」

「うまくいかなくても、 **1つ仮説が潰れたからOK** 」

「記録が取れていれば、検証材料になる」

といった感じになります。こういった“研究者の視点”で日常の行動を眺められるようになると、どんな仕事も成長のためのデータセットになりますんで、ぜひお試しください。

方略を実践に移す方法はいろいろあるんだけど、代表的なのは大きく4つです。

1. **「今日の方略宣言」を朝に書く、または声に出す** ：たとえば、「今日は“構成を先に紙で書く”方略を試してみる」「いつもの“いきなりパワポ開く”は使わずにいこう」みたいな感じで、朝に書いたり声に出したりすることで、メタ認知を活性化させられますんで。
2. **実行中は「やり方」を実況してみる** ：たとえば、「今、自分は“調べすぎ”のループに入りそうになってる」「思ったより“構成を先に書く”のが気持ちいいかも」みたいに、静かに心の中でつぶやくか、ChatGPT・メモ帳に実況してみましょう。「何をしているか」に気づくことが、方略の要であります。
3. **「あえてうまくいかない方法」も一度試す** ：たとえば、「いつもなら先に調べるところを、 **調べずに書き始めてみる** 」や「あえて“自分が避けがちな方法”をあえて実験する」みたいなところを意識する方法です．これにより、自分の“方略の前提”が明確になり、進化のヒントが生まれるんで、こちらもやっておきたいところ。
4. **実行中の「違和感」を記録する（感情ではなく身体感覚で）** ： 「この方法、気持ちよく進むな（集中感）」や「なんかモヤモヤする（手が止まる／迷いが多い）」みたいに、身体感覚の違和感をセルフモニタリングするのも重要っすね．ここで得られる「違和感メモ」が、次のステップ3「振り返り」に生きてきますんで。

ここらへんを意識しつつ、自分なりに“方略”の実験をしていただければと思うわけです。その手順を簡単にまとめておくと、

1. 今日の実験テーマを決める（「今日は「先に構成を書く」方略で資料を作ってみるぞ！」みたいに）
2. 実行中の観察メモを取る（「スライドに入るまでに時間がかかったが、迷いは少なかった」「書き出しの段階で、全体像を何度も見直せた」「いつもより集中が長く続いた気がする」みたいな感じで。
3. 実行中の違和感・気づいたクセもメモを取る（「つい過去資料に逃げたくなった」「書きすぎて構成が膨らみすぎた」みたいな感じで。
4. 最後に明日につなげたいこともメモしておく（「明日は、構成を3段落だけに制限してから作業に入ろう」みたいな感じで）

みたいになりまして、やってみると意外と簡単ですんで。

## ステップ3　振り返って、良かった点／微妙だった点を記録する

最後に、ここまでの振り返りをしておきましょう。ここで行うのはただの“反省”ではなくて、「自分の方略のパフォーマンス分析」であります。

1. 何が有効だったか？
2. 何が引っかかったか？
3. どう改善すれば、もっと自分に合ったやり方になるか？

この視点から検証していくことで、「メタ学習ループ」の質が上がっていきますんで。

このステップが大事な理由は、人間の脳は「実行」だけでは変化しないからです。私たちの脳ってのは、「意味づけ」された経験だけが記憶として残るようになってまして、うまくいったときに振り返らないと再現性のある成功パターンが見えなくなっちゃうんですよ。なので、「プロセスを記録する習慣」は、脳に記憶を残すために欠かせないんですな。

効果的な振り返りを行うためには、大きく5つの手順で進めていただくといいでしょう。

### 3-1「うまくいった点（良かった点）」を列挙する

次の質問を自分に投げかけて、「うまくいった戦略」を意識して次も使えるようにしましょう。

- 今日は、どの工程がスムーズだった？
- どのやり方が自分に合っていた？
- 集中できた瞬間はどこ？きっかけは？
- 「またやりたい」と思えた行動は？

### 3-2「引っかかった点（微妙だった点）」を列挙する

続きまして、以下の質問を自分に投げかけて、「引っかかった点（微妙だった点）」を意識してみましょう。

- 手が止まったのはどこ？
- 途中で集中が切れたのはなぜ？
- どこで時間がかかりすぎた？
- 何をしているときに“違和感”があった？

これによって方略の“相性”や“落とし穴”を見抜き、「改善の糸口」を見つけてくださいませ。

### 3-3 “意外な気づき”をメモする

以下の質問を自分に投げかけて、予想と違った結果や思わぬ手応えなどを掘り下げましょう。仮説検証の“外れ値”を拾って、新たな発見につなげるステップであります。

- 「構成を先に書いたら逆に固まってしまった」
- 「文章を書き始めたら集中が切れずに2時間経ってた」
- 「タイマー使ったら焦りが強くて逆効果だった」

### 3-4 “自分語り”でラベルをつける

続きまして、自分がやった「行動」に自分なりの名前をつけてみて、抽象化 → 記憶 → 再利用のプロセスを活性化させましょう。たとえば、

- 「スライド作成は“見切り発車→後から調整”型の方が自分向き」
- 「“リサーチ30分しばり”戦略は、脳が軽くなる感じ」
- 「“心配避けの過剰準備”が逆に集中を邪魔してるかも」

みたいな感じです。これによって行動の意味づけを強化し、次の意思決定の判断軸にできますんで。

### 3-5「次回の一手」を書く

最後に、ここまでの気づきをもとに、次回に使う“方略”の仮説を1つだけ決めておきましょう。たとえば、

- 明日は「●●戦略」を使ってやってみる
- 今日はやらなかったけど、「▲▲の方法」を試してみたい

みたいな感じでして、メタ学習ループの“再始動”をセットしておきましょう。

ということで、「振り返り」の王道はこんな感じですけども、 ここまでやるのが面倒だという人は、 最低でも以下ぐらいはメモしといてください。メモに使うものは、Slack・LINE・Notion・紙のふせん、何でもOKであります。

- 今日のやり方で：
	- うまくいったこと：＿＿＿＿＿＿
	- イマイチだったこと：＿＿＿＿＿＿
	- 明日どう変える？：＿＿＿＿＿＿

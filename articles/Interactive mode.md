## 概要

Claude Codeのインタラクティブモードに関する公式ドキュメント。ターミナル上でClaude Codeを使用する際のキーボードショートカット、テキスト編集、Vimモード、組み込みコマンド一覧、バックグラウンドタスク実行、プロンプト提案機能、サイドクエスチョン（`/btw`）、タスクリスト、PRレビューステータス表示など、セッション操作に関するすべての機能を詳述している。

## 主要なトピック

### キーボードショートカット

ショートカットはプラットフォームやターミナルにより異なる。`?` キーで利用可能なショートカットを確認できる。

**macOSでのOption/Altキー設定**: `Alt+B`、`Alt+F`、`Alt+Y`、`Alt+M`、`Alt+P` 等を使うには、ターミナルでOptionをMetaキーとして設定する必要がある。

- **VS Code**: Settings → Profiles → Keys → Left/Right Option key を "Esc+" に設定
- **Terminal.app**: Settings → Profiles → Keyboard → "Use Option as Meta Key" をチェック
- **iTerm2**: Settings → Profiles → Keys → Left/Right Option key を "Esc+" に設定

#### 一般操作

| ショートカット | 説明 |
| --- | --- |
| `Ctrl+C` | 現在の入力またはテキスト生成をキャンセル |
| `Ctrl+F` | バックグラウンドエージェントを全停止（3秒以内に2回押して確認） |
| `Ctrl+D` | セッション終了（EOF信号） |
| `Ctrl+G` | デフォルトテキストエディタで開く |
| `Ctrl+L` | ターミナル画面クリア（会話履歴は保持） |
| `Ctrl+O` | 詳細出力の切替（ツール使用状況の表示） |
| `Ctrl+R` | コマンド履歴の逆検索 |
| `Ctrl+V` / `Cmd+V` / `Alt+V` | クリップボードから画像を貼り付け |
| `Ctrl+B` | 実行中タスクをバックグラウンドへ移動（tmuxユーザーは2回押す） |
| `Ctrl+T` | タスクリストの表示/非表示切替 |
| `Esc` + `Esc` | 巻き戻しまたは要約（コード/会話を以前の状態に復元） |
| `Shift+Tab` / `Alt+M` | 権限モード切替（Auto-Accept / Plan / Normal） |
| `Option+P` / `Alt+P` | モデル切替 |
| `Option+T` / `Alt+T` | 拡張思考モードの切替 |

#### テキスト編集

| ショートカット | 説明 |
| --- | --- |
| `Ctrl+K` | カーソルから行末まで削除 |
| `Ctrl+U` | 行全体を削除 |
| `Ctrl+Y` | 削除したテキストを貼り付け |
| `Alt+Y` | 貼り付け履歴を巡回（`Ctrl+Y` の後） |
| `Alt+B` | 1単語分カーソルを戻す |
| `Alt+F` | 1単語分カーソルを進める |

#### マルチライン入力

| 方法 | ショートカット | 備考 |
| --- | --- | --- |
| クイックエスケープ | `\` + `Enter` | 全ターミナルで動作 |
| macOSデフォルト | `Option+Enter` | macOSデフォルト |
| Shift+Enter | `Shift+Enter` | iTerm2, WezTerm, Ghostty, Kittyで設定不要 |
| 制御シーケンス | `Ctrl+J` | ラインフィード文字 |
| ペーストモード | 直接貼り付け | コードブロック、ログ向け |

VS Code、Alacritty、Zed、Warpでは `/terminal-setup` で Shift+Enter のバインディングをインストールする必要がある。

#### クイックコマンド

| ショートカット | 説明 |
| --- | --- |
| `/` （行頭） | コマンドまたはスキル |
| `!` （行頭） | Bashモード（直接コマンド実行） |
| `@` | ファイルパスのオートコンプリート |

### Vimエディタモード

`/vim` コマンドまたは `/config` で恒久的に有効化可能。

**モード切替**: `Esc`（NORMALへ）、`i`/`I`/`a`/`A`/`o`/`O`（INSERTへの各種モード）

**ナビゲーション（NORMALモード）**:
- `h`/`j`/`k`/`l` — 左/下/上/右移動
- `w`/`e`/`b` — 次/末尾/前の単語
- `0`/`$`/`^` — 行頭/行末/最初の非空白文字
- `gg`/`G` — 入力の先頭/末尾
- `f{char}`/`F{char}`/`t{char}`/`T{char}` — 文字ジャンプ
- `;`/`,` — 最後のf/F/t/Tモーションの繰り返し/逆方向

**編集（NORMALモード）**:
- `x`/`dd`/`D` — 文字削除/行削除/行末まで削除
- `dw`/`de`/`db` — 単語の削除
- `cc`/`C`/`cw`/`ce`/`cb` — 変更操作
- `yy`/`Y`/`yw`/`ye`/`yb` — ヤンク（コピー）
- `p`/`P` — ペースト（後/前）
- `>>`/`<<` — インデント/デインデント
- `J` — 行結合、`.` — 最後の変更を繰り返し

**テキストオブジェクト**: `d`/`c`/`y` と組み合わせて `iw`/`aw`（単語）、`i"`/`a"`（引用符）、`i(`/`a(`（括弧）等が使用可能。

### コマンド履歴

- 履歴展開（`!`）はデフォルトで無効
- Up/Down矢印キーでナビゲーション
- `/clear` 実行時に入力履歴がリセット（以前のセッション会話は保持・再開可能）
- 入力履歴は作業ディレクトリごとに保存
- **Ctrl+R 逆検索**: 入力中のテキストで過去のコマンドをインクリメンタル検索。`Enter` で即実行、`Tab`/`Esc` で受け入れて編集継続

### バックグラウンドBashコマンド

長時間実行コマンドをバックグラウンドで実行し、作業を継続できる。

**使い方**:
- `Ctrl+B` で実行中のBashコマンドをバックグラウンドへ移動
- プロンプトでClaude Codeにバックグラウンド実行を指示

**特徴**:
- Claude Code終了時に自動クリーンアップ
- 一意のIDで追跡・出力取得可能（TaskOutputツール）
- 無効化: `CLAUDE_CODE_DISABLE_BACKGROUND_TASKS=1`

**よく使うバックグラウンドコマンド**: Docker、Terraform、開発サーバー、テストランナー（jest, pytest）、パッケージマネージャー、ビルドツール

**Bashモード（`!` プレフィクス）**: `! npm test` のようにClaude経由せず直接コマンド実行。履歴ベースのオートコンプリート、リアルタイム出力表示、会話コンテキストへの追加をサポート。

### プロンプト提案

セッション開始時、プロジェクトのgit履歴に基づいたグレーアウトされた提案コマンドが表示される。応答後も会話履歴に基づく提案が継続して表示される。

- 入力開始で消去、`Tab` で受け入れ、`Enter` で受け入れ即送信
- プロンプトキャッシュを再利用するため追加コストは最小限
- キャッシュが冷えている場合は生成をスキップ
- 非インタラクティブモード・プランモードでは自動スキップ
- 無効化: `CLAUDE_CODE_ENABLE_PROMPT_SUGGESTION=false`

### サイドクエスチョン（/btw）

`/btw` で会話履歴に追加せずに素早い質問が可能。現在の会話全体を参照できるが、エフェメラル（一時的）な表示。

**特徴**:
- **低コスト**: プロンプトキャッシュを再利用
- **単一応答**: フォローアップターンなし
- **ツールアクセスなし**: コンテキスト内の情報のみで回答
- **作業中も利用可能**: Claude処理中に独立して実行可能

**サブエージェントとの対比**: `/btw` は完全な会話を参照するがツールなし、サブエージェントは完全なツールを持つが空のコンテキストから開始。

### タスクリスト

複雑なマルチステップ作業時にClaudeがタスクリストを作成し、進捗を追跡。

- `Ctrl+T` で表示/非表示（最大10タスク表示）
- コンテキストコンパクション後も持続
- 旧TODOリストに戻す: `CLAUDE_CODE_ENABLE_TASKS=false`
- セッション間共有: `CLAUDE_CODE_TASK_LIST_ID=my-project claude`

### PRレビューステータス

ブランチにオープンなPRがある場合、フッターにクリック可能なPRリンクが表示される。

| 色 | ステータス |
| --- | --- |
| 紫 | マージ済み |
| グレー | ドラフト |
| 赤 | 変更要求 |
| 黄 | レビュー待ち |
| 緑 | 承認済み |

- `Cmd+click`（Mac）/ `Ctrl+click`で PRをブラウザで開く
- 60秒ごとに自動更新
- `gh` CLIのインストールと認証が必要

## 重要な事実・データ

- **構文ハイライト**: Claude Codeのネイティブビルドのみで利用可能
- **プロンプト提案コスト**: プロンプトキャッシュを再利用するため追加コストは最小限
- **PRステータス更新間隔**: 60秒ごと
- **タスクリスト表示上限**: 最大10タスク
- **Vim Normal モード**: カーソルが入力の先頭/末尾にあり移動できない場合、矢印キーでコマンド履歴をナビゲート

## 結論・示唆

### 実践的な示唆

- ターミナル環境に応じてOption/Metaキーの設定が必要（特にmacOS）
- `/terminal-setup` コマンドで不足するキーバインディングを自動インストール可能
- `/btw` とサブエージェントを使い分けることで、効率的な情報取得が可能
- バックグラウンドタスクと`!` Bashモードを活用して、Claude Codeセッション中の生産性を最大化
- タスクリストの `CLAUDE_CODE_TASK_LIST_ID` でプロジェクト間のタスク共有が可能

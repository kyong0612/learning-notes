## 概要

Claude Code の `.claude/rules/` ディレクトリを使用して、ルールファイルを分割管理する手法についてのスレッド。単一の巨大な `CLAUDE.md` ファイルを維持するのではなく、コンテキストごとにルールを分割することで、管理性と効率を向上させるアプローチを紹介している。

## 主要なポイント

### `.claude/rules/` ディレクトリの機能

- **肥大化の防止**: `.claude/rules/` ディレクトリ内に複数の `.md` ファイルを作成することで、`CLAUDE.md` の肥大化を防ぐことができる
- **コンテキスト依存のルール読み込み**: 作業中のファイルに関連するルールだけが読み込まれる仕組み
- **セキュリティルールの全体適用**: セキュリティに関するルールは全体に適用される
- **分割整理**: 指示を複数のファイルに分割して整理可能

### コンテキスト管理のメリット

- 巨大な1ファイルより、文脈ごとにルールを分割する方がAIも人間も迷わない
- ルールを整理することで管理しやすくなる

## 関連情報

この機能は Claude Code（Anthropic社のCLIツール）で利用可能。プロジェクトルートに `.claude/` ディレクトリを作成し、その中の `rules/` サブディレクトリにMarkdownファイルを配置することで、ルールを分割管理できる。

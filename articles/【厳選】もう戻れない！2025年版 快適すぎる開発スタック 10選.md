## 概要

2025年、著者の開発者体験（DX）は大きく進化した。SQL、`'use client'`の分断、認証実装、Stripeの税金地獄といった「やりたくないこと」から解放される時代が来ている。

**本記事の主張**: 良いDXは良いUXを生み出す。開発者の心の余裕が、クリエイティビティが、ユーザーのことを考える余裕と素晴らしい体験を生む。

---

## 1. TanStack Start

**提言**: Next.jsに苦しめられるのをやめませんか？

### 問題点
- Next.jsはパフォーマンス追求には最高だが、Overspecになりがち
- ディレクティブの増殖問題（`'use client'`、`'use server'`、`'use cache'`）
- `'use client'`により本来1つで良いコンポーネントを2つに分ける必要がある
- Server Componentのfetchでは型がつかない
- Route の型安全性が不完全（SearchParams等は外部ライブラリが必要）

### TanStack Startの優位性
- **ディレクティブの概念が存在しない**
- TypeScript 100%サポートで型安全が最初から担保
- TanStack Routerによる完全な型安全ルーティング
- 作者のTanner Lisleyがディレクティブの増殖に対して懸念を表明

> Server Functions - Type-safe RPCs between client and server
> TanStack Start relies 100% on TanStack Router for its routing system.

## 2. Convex

**提言**: SQL書くのをやめませんか？

### 概要
Convexはオープンソースのリアクティブデータベースで、認証・Function・Cron・Storageなどバックエンドに必要なソリューションが揃ったBaaS。

### Supabaseとの比較
| 項目 | Supabase | Convex |
|------|----------|--------|
| スキーマ定義 | SQL | TypeScript（宣言的） |
| リアルタイム | 明示的に有効化必要 | デフォルトで搭載 |
| SQL記述 | 必要 | **不要** |

### コード例
```typescript
// Convexでのスキーマ定義
import { defineSchema, defineTable } from "convex/server";
import { v } from "convex/values";

export default defineSchema({
  tasks: defineTable({
   text: v.string(),
   isCompleted: v.boolean(),
 }),
});
```

### DX中心の設計
- 環境変数ファイルは自動生成
- SchemaをDBに自動反映
- FunctionやCronも自動反映
- ファイル更新時に内容も自動反映

> Convex is a "document-relational" database. JSONも扱えてIDによるRelationも持てる

## 4. ElysiaJS

**提言**: OpenAPI生成で苦しむのをやめませんか？

### 概要
- Bun上で動作する高性能なTypeScript Webフレームワーク
- 「JS/TS界隈のFastAPI」と呼ばれる

### OpenAPI対応
ElysiaJS自体がOpenAPIを採用しており、**1行追加するだけでOpenAPIドキュメント生成可能**：

```typescript
import { Elysia, t } from 'elysia'
import { openapi } from '@elysiajs/openapi'

new Elysia()
    .use(openapi())  // この1行だけ
    .get('/user/:id', ({ params: { id } }) => id, {
        params: t.Object({
            id: t.Number()
        })
    })
    .listen(3000)
```

### 追加のメリット
- 型からOpenAPIスキーマを生成可能（`fromTypes()`）
- Bunネイティブなのでパフォーマンス良好
- マルチランタイム対応

## 5. Polar.sh

**提言**: Stripeの税金地獄から解放されませんか？

### 概要
ソフトウェア開発者向けの決済プラットフォーム。**Merchant of Record（MoR）**として、グローバルな税務コンプライアンスを代行。

### 最大のメリット
- **国際的な税務コンプライアンスをすべて管理**
- 世界中で税金の計算、徴収、納税を行う
- 開発者はプロダクト開発に集中できる

### Stripeとの比較
- Polar内部ではStripeを使用（信頼性担保）
- Stripeの上に構築された「MoR + 請求 + エンタイトルメント」レイヤー
- DXでStripeの完全上位互換
- **人間工学に基づいた直感的な開発**（Stripeは直感と反することが多い）

## 7. Cursor

**提言**: CLI・IDE・Desktopの往復を終わりにしませんか？

### 概要
AI補完をネイティブに統合したコードエディタ。

### Cursor 2.0以降の主要機能

#### Composer
- 同等の知性を持つモデルと比べて**4倍速い**エージェント型コーディングモデル
- **マルチエージェント機能**: 1つのプロンプトで最大8つのエージェントを並列実行
- 各エージェントがコードベースの独立したコピー上で動作

#### AIコードレビュー

- GitHubやGitLabのBugbotと連携

#### ブラウザー機能
- エディタ内にブラウザを埋め込み

#### ボイスモード

- promptの入力は音声入力が最速

## 8. ni

**提言**: パッケージマネージャーを意識するのをやめませんか？

### 概要
ロックファイル（`package-lock.json`、`yarn.lock`、`pnpm-lock.yaml`、`bun.lock`等）を自動検出して、適切なパッケージマネージャーでコマンドを実行するツール。

### 使用例
```bash
# どのプロジェクトでも同じコマンド
ni vite
# → npmプロジェクトなら npm i vite
# → yarnプロジェクトなら yarn add vite
# → pnpmプロジェクトなら pnpm add vite
# → bunプロジェクトなら bun add vite

nr dev --port=3000  # npm run dev / yarn run dev / etc.

nun webpack  # npm uninstall / yarn remove / etc.
```

### 追加機能
- `ni -i`: インタラクティブにパッケージを検索してインストール
- `nr`: スクリプトを選択実行

## 10. Resend

**提言**: メールプロバイダー選択とメールHTML地獄から解放されませんか？

### 概要
開発者向けのメール送信API。**React Email**との組み合わせでDXを劇的に向上。

### React Emailの利点
- Reactコンポーネントでメールテンプレートを作成
- TypeScriptで型安全にメールを作成
- 主要メールクライアント（Gmail、Apple Mail、Outlook等）をサポート

### コード例
```tsx
// React Emailでメールテンプレートを作成
import { Html, Body, Container, Text, Button } from '@react-email/components';

export function WelcomeEmail({ name, url }: { name: string; url: string }) {
  return (
    <Html>
      <Body>
        <Container>
          <Text>こんにちは、{name}さん！</Text>
          <Button href={url}>アカウントを有効化</Button>
        </Container>
      </Body>
    </Html>
  );
}
```

### ツール機能

#### SpamAssassin統合
- メールプロバイダーによってスパムとしてマークされる可能性を予測
- 過度な大文字使用、問題のあるフレーズ、隠しテキストなどをフラグ付け

#### Linter機能
- メールリンクがHTTPSを使用し、有効であることを確認
- 画像にaltタグがあり、1MB以下のサイズ制限内であることを確認
- 互換性チェッカーがHTMLとCSSを分析

### new.email
- Resendのメールテンプレート生成AI
- promptを打てばReact Emailの形式でテンプレート作成（v0のメール版）

### React Email v5の新機能
- **Resend Templates**: チームでメールを共同編集（Figmaのような体験）
- Tailwind v4対応
- 8つの新コンポーネント追加

## 【番外編】 React Compiler Marker

**提言**: Reactコンポーネント最適化に悩むのをやめませんか？

### 概要
VSCode/Cursor向けの拡張機能。React Compilerによって最適化されたコンポーネントを視覚的にマーキング。

### 機能
- **✨マーク**: 最適化されたコンポーネント
- **🚫マーク**: 最適化されなかったコンポーネント（ツールチップで理由表示）

### メリット
- **即座にフィードバック**: コードを書きながらリアルタイムで最適化状況が分かる
- **学習ツール**: なぜ最適化されないのかツールチップで理由が分かる
- **反復的な改善**: コード変更後、マークが🚫から✨に変わるのを確認可能

## 結論

> TypeScriptを知ってしまったらJavaScriptには戻れないように、これらの次世代開発者体験を知ってしまったら、もう元には戻れない

本記事で紹介した10個の技術スタック・ツールは、すべて「**面倒なことを手放す**」という共通の思想を持っている。開発者が本来集中すべき「クリエイティブな部分」に時間を使えるよう、煩雑な作業を自動化・抽象化してくれる。

**DXが良ければUXも良くなる**。開発者の心の余裕が、ユーザーのことを考える余裕を生み、結果として素晴らしい体験を生み出す。

## 概要

本プレゼンテーションは、2025年10月10日に開催された「UI/UXも、AIと共に。～フロントエンド開発におけるAI活用事例～」イベントにおける発表資料です。Figma MCPとKombaiという最新のAIツールを用いて、Figmaデザインをどこまで忠実にコード化できるかを実験的に検証しています。

**Speaker**: Oikon（[@oikon48](https://x.com/oikon48)）  
**関連イベント**: [https://rosca.connpass.com/event/369090/](https://rosca.connpass.com/event/369090/)

## UI/UXデザイン実装の現状と課題

### デザインからコードへの変換における主な課題

1. **再現度**
   - デザインの意図を正確にコード化することが難しい
   - AI特有の「AI臭のあるデザイン」になりがち

2. **実装面**
   - 明確な指示がないとレスポンシブ対応が抜けやすい

3. **知識面**
   - ライブラリやフレームワークの最新情報を保持していない

4. **保守性**
   - 一貫性のあるコンポーネント設計が困難

**現状の認識**:

- ピクセルパーフェクトな実装はまだ難しい
- Figma → Codeは比較的デザインに忠実

## Figma MCPについて

### Figma MCPの特徴

- **直接データアクセス**: FigmaのデザインデータをAIが直接読み取り可能
- **柔軟な指示**: プロンプトで細かい実装指示が可能
- **意図理解**: ClaudeやCodexがデザインの意図を理解

### Figma MCPの使用方法

1. Figma MCPをClaude Code/Codex CLIに接続
2. Figmaファイル/フレームのURLを指定
3. 自然言語でコード生成を指示

**設定ファイル**:

- `.claude.json` (Claude Code用)
- `~/.codex/config.toml` (Codex CLI用)

## Kombaiについて

### Kombaiの概要

インドのスタートアップが開発したフロントエンド専用AI Agentツール

### Kombaiの特徴

1. **プロジェクト全体の解析**
   - プロジェクト全体を分析して実装

2. **技術スタック設定**
   - フレームワーク・ライブラリを事前に設定可能

3. **安全な実装プロセス**
   - 仮想環境内で実行
   - Acceptするまでローカルに変更を加えない

## 実験: 3ツールの比較検証

### 検証方法

同一のFigmaデザインを使用し、以下の3つのツールで実装を比較

**比較対象ツール**:

1. Claude Code + Figma MCP
2. Codex CLI + Figma MCP
3. Kombai

**共通プロンプト**:

```text
FIGMA_DESIGN_URL このFigma Designを実装して
```

### 実験結果

各ツールで実装したコードの再現度を比較検証しました。（詳細な比較結果はスライド9-12に掲載）

## まとめ

### 主な知見

1. **再現度**: まだピクセルパーフェクトは難しいが、**忠実度70%くらいは再現できる**

2. **技術的整合性**: フレームワーク・ライブラリの整合性は、まだエンジニアによる評価が必要

### 今後の展望

AI駆動型のデザイン実装ツールは急速に進化しており、エンジニアとデザイナーの協働方法に大きな変化をもたらす可能性があります。現時点では完全な自動化には至っていませんが、開発効率の向上に貢献できるレベルに達しています。

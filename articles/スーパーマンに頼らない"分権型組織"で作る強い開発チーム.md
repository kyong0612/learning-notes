## 概要

スマートバンク サーバーサイド部 部長の三谷昌平氏（[@shohei1913](https://twitter.com/shohei1913)）が、EMConf JP 2026（2026/03/04）で発表したセッション。人が増えたのに生産性が上がらない問題に対し、**「委員会制度」という分権型の組織運営モデル**を導入して1年間取り組んだ成果と学びを共有している。

## 背景：人が増えたのに楽にならない問題

### Mission Team制度の導入（2024年）

- **2024年以前**: 短期解散のプロジェクトチーム中心。案件ごとにチームを組み直す柔軟な体制
- **2024年から**: 長期固定のMission Team制度へ移行。チームのナレッジを醸成し、リリース後も改善責任を中長期で担う
- 採用を強化し、PM/デザイナー/エンジニアの人数は約**1.5倍**に増加
- Mission Team制度により「カードの決済管理アプリ」から「家計簿アプリ」へと進化。ストーリー性のある機能開発が加速

### 顕在化した組織課題

Mission Teamで事業の重要施策に集中する一方、**Missionに位置付けられない重要課題**が取り残された：

1. **運用の「仕組み」不全**
   - ルールの周知漏れで作業品質がばらつく（新メンバーがルールを知らない、障害時の連絡先が漏れる）
   - オーナー不明確で障害対応が遅れる（「あの人が反応してるから大丈夫」と関与が弱まる）
   - 以前は問題なかった配信バッチや管理画面操作でDB負荷が高まり障害に

2. **保守運用課題の優先順位問題**
   - MT開発と保守運用の配分を8:2と決めるが判断が困難
   - 重要だが緊急でない横断タスクが後回し → 気づかないうちに障害点として影響が拡大
   - 善意や自主性に任せすぎることのマネジメントリスク

3. **全員野球の限界**
   - パフォーマンスMTGや部定例への参加者が増え続け、発言機会が減少
   - 人が多くなるほど意思決定の質とスピードが低下

### 本質的な問題

> 暗黙知や主体性頼りの組織の限界が見え始めた。人数増に横串の組織戦略が追いついていない。

## 目指した姿：3つの"したい"

1. 本当に重要な課題を見極めて、**先手を打って解決したい**
2. 任せられるところは任せて、**自分がやるべきことに集中したい**
3. 問題に追われるのをやめて、**楽しくエンジニアリングしたい**

## 委員会制度の設計

### 基本定義

- **委員会** = 事業成長を妨げる重要課題に取り組む少人数チーム
- 課題解決の**実行と結果の責任**を負う
- 何の課題をどう解決するか決める**権限**も持つ
- 必ず1名の**委員長**を任命し、やり切る責任を負う

### 3つのコンセプト

#### コンセプト1: 横断課題の"器"を作る

- EM陣で部の最重要課題を徹底議論し抽出
- **4つの課題領域**を設定：

| 課題領域 | 委員会 | 内容 |
|----------|--------|------|
| サービスの品質の維持・向上 | 品質委員会 | SLOの設定と監視、CIの高速化 |
| 将来必要な技術領域への投資 | 投資委員会 | AI活用の環境整備 |
| 技術発信と採用力強化 | 広報委員会 | ブログ、登壇、スポンサー |
| 保守運用業務の負担軽減 | チーム連携最大化委員会 | 手順書整備、アサイン |

- **活動サイクルは半年**：変化に合わせやすく、"今"最重要な課題にフォーカス
- 下期には**パフォ活委員会・DevEx委員会・Ops委員会・データ活用委員会・広報委員会**に再編

#### コンセプト2: 権限委譲と解決スピード

- **RACIチャート**で権限を明確化：

| RACI | 役割 | 担当 |
|------|------|------|
| Responsible（実行責任者） | 委員会に所属する委員 | 委員 |
| Accountable（説明責任者） | 委員長 | 委員長 |
| Consulted（相談先） | サーバーサイド部全員 | 部員 |
| Informed（報告先） | 部長 | 部長 |

- 委員会の役割は**抽象度高く設定**（期中の様々な課題を柔軟に解決するため）
- 5W1Hは全て自分たちが決定可能。部長のオーダーも取り入れるかは委員会が決める
- **分権型の思想**: 組織が大きくなるほどEMが全てを把握するのは不可能。問題に近い人の方が解像度が高く適切な意思決定がしやすい

#### コンセプト3: 人材育成

- 委員長の意思決定は部全体に影響（障害のサービスへの被害、半年後の生産性）
- 意思決定の機会を増やすことで成長する場を提供
- Mission Teamの垣根を超えたエンジニア間の交流を促進

### 他社事例

- **Spotify**: ギルド（有機的で広範な関心のあるコミュニティ。知識・ツール・コード・実践を共有）
- **ピクシブ**: エンジニアギルド（採用・評価・育成を事業部横断で改善する仕組み）
- メルカリ / Nstock / ユーザベース / ギフティなども類似の取り組み
- スマートバンクは委員会とメンバーを**トップダウンで決定**する点が特徴

## 委員会が生み出した成果

| 成果 | 詳細 |
|------|------|
| **DB負荷の根本対応** | Primary 1つ運用からリードレプリカ導入を決定。170以上のAPIの対応を完了 |
| **やばいよやばいよメトリクス** | 24H超えバッチの検出と計画的対応の仕組み化 |
| **Runbook整備** | 保守運用の手順書作成。実施日・作業内容・時間を蓄積し、次回対応や自動化のインプットに |
| **CI高速化** | 10分以上かかるCIを改善。Self Hosted Runner導入で**実行時間は半分、GHAの運用費は1/3に削減** |
| **AI活用推進** | AIサービス利用ガイドライン整備、開発者向けAIツール予算策定、Devinの活用浸透 |
| **データ分析Agent（Ask）** | 社内向けAIアシスタント。AI Agentがクエリを作成・実行。**社内の75%以上が活用** |
| **登壇・発信** | 2025年は**46イベントで登壇**、6イベントでスポンサー、新春エンジニア駅伝ブログ |

## 委員会をうまく回す工夫

### 活動しやすい環境づくり

- **全社定例で実績をアピール**し、活動意義と重要性を社内に共有
- Mission Teamリーダーへは**相手目線でのメリット**を伝える：
  - 「デプロイが定期的に失敗する」→「MTの施策が出したいときに出せない状況を解決している」
  - 「AIのガイドラインを作っている」→「AI活用推進でMTの開発スピードが早くなる」

### 次世代リーダーの育成

- **グレードや年次に関係なく委員長に抜擢**
- Mission Teamでは役割が固定されがちだが、委員会で様々なポジションを経験する機会を提供
- シニアエンジニアをまとめたり意思決定が求められる絶好の成長機会

### 予算マネジメント

- **予算のトラッキングシートを作成**：毎月実績を更新、定期的に経営陣と交渉
- お金の使い方に消極的になりがちなエンジニアに対し、使える金額感をオープンに
- お金の相談をされたとき嫌な顔をしない。むしろ背中を押す

### 楽しい環境を作る

- 新しい技術を試したい人の背中を押す（Self Hosted Runner、Cloudflare、AI Agent）
- 「いいじゃん」「面白そう」「やってみようよ」という姿勢
- 委員会の活動が個人のスキルやキャリアを伸ばすことにつながることを重視

## EMとしての学び

### 委員会は "強制的な遊び場"

- 課題解決を自主性の高さに任せすぎない
  - 視座やオーナシップが高い「スーパーマン」もいるが、「やりたいけどやっていいのか分からない」と待っている人もいる
  - 自主的にやりたいこと ≠ 組織が必要としていることばかりではない
- **全員の力を合わせるには強制力という"仕掛け"が絶対に必要**
- 責任を持ってもらいつつ、クリエイティビティが発揮されるような環境を作ることが大事

### 任せることの価値

- どの課題領域でも信頼して任せられるメンバーがいることは幸せ
- 任せることでその人のスキルや性格の理解度も上がる
- 任せずにマネージャーだけで抱えていたら、多くても**1/5くらいしか解決できなかった**と推測

### 残る課題

- 委員会は**銀の弾丸ではない**
- Mission Teamの佳境タイミングでは委員会の動きが遅くなる
- 委員長には変数が多いProject Managementが求められる
- 課題によっては委員会とは別の**常設組織を作る方が良い**場合もある

## 結論

> 人や組織がうまく回っていないところを見つけて、治水工事のように流れを整えていくところがEMの楽しさ。隣でみんなが楽しく開発しているのを見ると嬉しい。

**委員会制度のポイント**：
- 横断課題の"器"を戦略的に設計する
- 権限と責任をセットで委譲し、意思決定スピードを上げる
- 「スーパーマン」に頼らず、組織の仕組みで全員の力を引き出す
- 強制力のある仕掛けの中で、個人の成長とクリエイティビティを促進する

## 概要

[microgpt](https://gist.github.com/karpathy/8627fe009c40f57531cb18360106ce95) は、Andrej Karpathyによるアートプロジェクトで、**依存関係なしの純粋なPython 200行**でGPTの学習と推論を完結させる単一ファイルである。データセット、トークナイザー、自動微分エンジン、GPT-2風ニューラルネットワークアーキテクチャ、Adamオプティマイザー、学習ループ、推論ループの全アルゴリズムを含む。micrograd、makemore、nanogptなどの複数プロジェクトと10年に及ぶLLM簡素化への追求の集大成とされる。

## 1. Dataset（データセット）

LLMの燃料はテキストデータのストリーム。本番ではWebページだが、microgptでは **32,000件の名前**（1行1名前）をデータセットとして使用する。

```python
docs = [l.strip() for l in open('input.txt').read().strip().split('\n') if l.strip()]
random.shuffle(docs)
```

モデルの目標は、データのパターンを学習し、同様の統計的パターンを持つ新しい「ドキュメント」を生成すること。学習後、モデルは「kamon」「karai」「vialan」のような、もっともらしい新しい名前を生成（"幻覚"）できる。

ChatGPTの視点からも、ユーザーとの会話は単なる「ドキュメント」であり、モデルの応答は統計的なドキュメント補完にすぎない。

---

## 2. Tokenizer（トークナイザー）

ニューラルネットワークは数値で動作するため、テキストを整数トークンIDのシーケンスに変換する必要がある。本番のトークナイザー（tiktoken等）は文字チャンク単位で動作するが、最も単純なトークナイザーは**各ユニーク文字に1つの整数を割り当てる**。

- データセット内のユニーク文字（a-z）をソートし、インデックスでIDを付与
- **BOS（Beginning of Sequence）**トークンを特別トークンとして追加（区切り文字として機能）
- 各ドキュメントはBOSで囲まれる: `[BOS, e, m, m, a, BOS]`
- 最終語彙サイズ: **27**（a-zの26文字 + BOSトークン1つ）

## 3. Autograd（自動微分）

学習には勾配が必要: 各パラメータについて「この数値を少し上げたら、損失は上がるか下がるか、どれくらいか？」を知る必要がある。逆伝播は連鎖律（chain rule）に基づく。

### Valueクラス

`Value`クラスがスカラー値の自動微分を実装する:

- `data`: フォワードパスで計算されたスカラー値
- `grad`: バックワードパスで計算された損失に対する導関数
- `_children`: 計算グラフにおける子ノード
- `_local_grads`: 各子ノードに対する局所導関数

### サポートされる演算と局所勾配

| 演算 | 順伝播 | 局所勾配 |
|------|--------|----------|
| `a + b` | \(a + b\) | \(\frac{\partial}{\partial a} = 1, \frac{\partial}{\partial b} = 1\) |
| `a * b` | \(a \cdot b\) | \(\frac{\partial}{\partial a} = b, \frac{\partial}{\partial b} = a\) |
| `a ** n` | \(a^n\) | \(\frac{\partial}{\partial a} = n \cdot a^{n-1}\) |
| `log(a)` | \(\ln(a)\) | \(\frac{\partial}{\partial a} = \frac{1}{a}\) |
| `exp(a)` | \(e^a\) | \(\frac{\partial}{\partial a} = e^a\) |
| `relu(a)` | \(\max(0, a)\) | \(\frac{\partial}{\partial a} = \mathbf{1}_{a > 0}\) |

### backward()メソッド

グラフを逆トポロジカル順に走査し、各ステップで連鎖律を適用:

\[\frac{\partial L}{\partial c} \mathrel{+}= \frac{\partial v}{\partial c} \cdot \frac{\partial L}{\partial v}\]

**連鎖律の直感的理解**: 「車が自転車の2倍速く、自転車が徒歩の4倍速いなら、車は徒歩の2×4=8倍速い」。連鎖律は変化率を経路に沿って掛け合わせるだけ。

**勾配の累積（+=）**: ある値がグラフ内の複数箇所で使用される場合（分岐）、各分岐からの勾配は独立に逆流し合算される。多変数連鎖律の帰結。

PyTorchの`loss.backward()`と同じアルゴリズムだが、テンソルではなくスカラーで動作する。アルゴリズム的に同一だが、効率は大幅に劣る。

## 4. Parameters（パラメータ）

パラメータはモデルの知識であり、ランダムな浮動小数点数として初期化され、学習中に最適化される。

### モデル構成

- `n_embd = 16` — 埋め込み次元
- `n_head = 4` — アテンションヘッド数
- `n_layer = 1` — レイヤー数
- `block_size = 16` — 最大シーケンス長

### パラメータ行列

| パラメータ | サイズ | 役割 |
|-----------|--------|------|
| `wte` | vocab_size × n_embd | トークン埋め込み |
| `wpe` | block_size × n_embd | 位置埋め込み |
| `lm_head` | vocab_size × n_embd | 出力射影 |
| `attn_wq/wk/wv/wo` | n_embd × n_embd | アテンション重み |
| `mlp_fc1` | 4×n_embd × n_embd | MLP第1層 |
| `mlp_fc2` | n_embd × 4×n_embd | MLP第2層 |

合計パラメータ数: **4,192**（GPT-2は16億、現代のLLMは数千億）

## 5. Architecture（アーキテクチャ）

GPT-2をベースに以下を簡略化: LayerNorm→**RMSNorm**、バイアスなし、GeLU→**ReLU**。

### ヘルパー関数

- **`linear(x, w)`**: 行列-ベクトル積。ニューラルネットの基本構成要素（学習された線形変換）
- **`softmax(logits)`**: ロジットを確率分布に変換（全値が[0,1]で合計1）。最大値の減算で数値安定性を確保
- **`rmsnorm(x)`**: Root Mean Square正規化。活性化の大きさを安定させ、学習を安定化する（LayerNormの簡略版）

### GPT関数の処理フロー

1. **Embeddings（埋め込み）**: トークンIDと位置IDからそれぞれの埋め込みテーブルを参照し加算。モデルに「何」のトークンか、「どこ」の位置かの情報を提供
2. **Attention block（アテンションブロック）**: 
   - 現在のトークンをQuery(Q)、Key(K)、Value(V)に射影
   - Q=「何を探しているか」、K=「何を含んでいるか」、V=「選ばれた場合に何を提供するか」
   - KVキャッシュに追加し、過去のトークン情報にアクセス
   - マルチヘッドアテンション: 各ヘッドが独立にドット積→softmax→重み付き和を計算
   - **アテンションはトークン間通信メカニズム**（位置tが過去0..t-1を「見る」唯一の場所）
3. **MLP block**: 2層フィードフォワードネットワーク（4倍次元に拡大→ReLU→元に縮小）。各位置での「思考」を担当。アテンションと異なり完全にローカルな計算
4. **Residual connections（残差接続）**: 両ブロックの出力を入力に加算。勾配の直接流通を可能にし、深いモデルの学習を可能にする
5. **Output（出力）**: 最終隠れ状態を`lm_head`で語彙サイズに射影し、27個のロジットを生成

### KVキャッシュについて

通常は推論時のみと考えられるKVキャッシュだが、学習中も概念的には常に存在する。本番実装ではベクトル化されたアテンション計算内に隠れているだけ。microgptはトークンを1つずつ処理するため明示的に構築し、キャッシュ内のValue ノードを通じて逆伝播も行う。

## 6. Training Loop（学習ループ）

各ステップで: (1) ドキュメント選択 → (2) モデル順伝播 → (3) 損失計算 → (4) 逆伝播 → (5) パラメータ更新

### 処理の詳細

- **トークン化**: 各ドキュメントをBOSで囲む（例: "emma" → `[BOS, e, m, m, a, BOS]`）
- **損失関数**: クロスエントロピー損失 \(-\log p(\text{target})\)
  - 正しいトークンに確率1.0を割り当てれば損失0
  - 確率が0に近いほど損失は+∞に近づく
  - 各位置の損失を平均してスカラー損失を得る
- **Adamオプティマイザー**: 単純な勾配降下より賢い最適化
  - `m`: 最近の勾配の平均（モメンタム）
  - `v`: 最近の勾配の二乗の平均（パラメータごとの学習率適応）
  - バイアス補正（`m_hat`, `v_hat`）付き
  - 学習率は線形減衰

### 学習結果

- 損失は約3.3（ランダム推測: \(-\log(1/27) \approx 3.3\)）から約**2.37**に低下
- 最低値は0（完全予測）なので改善の余地はあるが、名前の統計的パターンを学習している

## 7. Inference（推論）

学習後、パラメータを固定して順伝播ループで新しい名前を生成:

1. BOSトークンから開始（「新しい名前を始めよ」）
2. 27個のロジットを確率に変換
3. 確率に従ってランダムにトークンをサンプリング
4. 生成したトークンを次の入力として戻す
5. BOSが出力されるか最大長に達するまで繰り返す

### Temperature

- ロジットをtemperatureで割ってからsoftmaxに通す
- **temperature=1.0**: モデルの学習済み分布から直接サンプリング
- **低い値（例: 0.5）**: 分布を鋭くし、トップ候補を選びやすくする（保守的）
- **0に近い値**: 常に最も可能性の高いトークンを選択（greedy decoding）
- **高い値**: 分布を平坦にし、多様だが一貫性の低い出力を生成

## 9. 本番LLMとの違い（Real stuff）

microgptはGPT学習・推論の完全なアルゴリズム的本質を含むが、本番LLM（ChatGPT等）との間には多くの差異がある。コアアルゴリズムと全体レイアウトは変わらないが、スケールで機能させるために必要なもの:

| 領域 | microgpt | 本番LLM |
|------|----------|---------|
| **データ** | 32K件の短い名前 | 数兆トークンのインターネットテキスト（重複排除、品質フィルタリング済み） |
| **トークナイザー** | 単一文字 | BPE（サブワード）、語彙サイズ約100K |
| **Autograd** | スカラーValue、純粋Python | テンソル、GPU/TPU、PyTorch、FlashAttention |
| **アーキテクチャ** | 4,192パラメータ | 数千億パラメータ、RoPE、GQA、MoE等 |
| **学習** | 1ドキュメント/ステップ | 大バッチ（数百万トークン/ステップ）、混合精度 |
| **最適化** | Adam + 線形減衰 | 精密なハイパーパラメータ調整、スケーリング則 |
| **後処理** | なし | SFT（教師あり微調整）+ RL（強化学習） |
| **推論サービング** | そのまま実行 | バッチ処理、KVキャッシュ管理、投機的デコーディング、量子化 |

## 10. FAQ

- **モデルは「理解」しているか？**: メカニズム的には魔法はない。入力トークンを次のトークンの確率分布にマッピングする大きな数学関数。「理解」に相当するかは哲学的な問題
- **なぜ機能するか？**: 数千の調整可能なパラメータを各ステップで微調整し損失を下げる。多くのステップを経てデータの統計的規則性を捉える値に落ち着く
- **ChatGPTとの関係**: 同じコアループ（次トークン予測→サンプリング→繰り返し）を大規模化し、会話型にする後処理を追加したもの
- **「幻覚」とは**: 確率分布からサンプリングするため真実の概念はなく、統計的にもっともらしいシーケンスを生成するだけ。microgptの「karia」生成もChatGPTの偽事実生成も同じ現象
- **なぜ遅いか**: 純粋Pythonで1スカラーずつ処理。GPUなら数百万スカラーを並列処理し桁違いに高速
- **より良い名前を生成するには**: `num_steps`増加、モデルサイズ拡大（`n_embd`, `n_layer`, `n_head`）、またはより大きなデータセットの使用
- **データセットを変えたら**: モデルはデータ内のパターンを学習する。都市名、ポケモン名、英単語、短い詩など何でも可能。コードの変更は不要

## 概要

**Pencil.dev**は、コードエディタ（Cursor、VSCode、Claude Code等）内に直接組み込まれるAIドリブンなデザインキャンバスツール。創業者はTom Krcha（元Around / Alter avatars）、a16z出資、2025年設立のカリフォルニア拠点スタートアップ。

従来のデザイン→コードのハンドオフ問題（Figmaで更新されたデザインとコードがズレる）を、**デザインファイルをGitリポジトリに直接格納**するアプローチで解消する。

## 主要機能

### 無限デザインキャンバス

- IDEを離れることなく、ピクセルパーフェクトなデザインが可能
- コードとデザインがワンクリックで切り替え可能

### AIシングルプレイヤー（＝新しいマルチプレイヤー）

- AIエージェントが並列で画面やフロー全体を生成
- 新しいデザイン方向を素早く探索できる「追加の手」として機能

### Vibe Designing（プロンプトベースのデザイン）

- 画面全体または特定パーツに対してプロンプトで直接指示
- 速度と品質に最適化されたキュレーション済みアクション、またはカスタムプロンプト

### ベクター→コード変換（ピクセルパーフェクト）

- デザインからHTML、CSS、Reactコードを本番品質で出力
- デザインとコードが同一リポジトリに共存

### デザインキット

- キュレーション済みのコンポーネントベースデザインキットを内蔵
- チームの既存デザインシステムをコードベースからプラグイン可能

## アーキテクチャの特徴

### MCP（Model Context Protocol）ファースト設計

Pencilの最大の差別化ポイントは、AnthropicのMCPを中心に設計されていること。

| 比較項目 | Figma MCP | Pencil MCP |
|---------|-----------|------------|
| アクセス | **読み取り専用** | **読み書き両方** |
| AI操作 | デザインを見るだけ | デザインを直接操作 |

創業者Tom Krchaの言葉：「すべてのAIエージェントがデザインファイルを読み書きできる」

実用例：
- キャンバスにフォームをスケッチ → AIが対応するHTML/CSSを生成
- コンポーネント間に矢印を引いて「クリックでモーダル表示」とラベル → AIがイベントバインディングを記述

従来の「デザイン→コード」ツールがデザインを**参照画像として解釈**するのに対し、Pencilはデザインを**構造化データとして直接操作**する。解釈は近似値を生み、直接操作は正確性を生む。

### Git ネイティブ

デザインファイルがオープンフォーマットでリポジトリに格納される。

- **コードをロールバック = デザインもロールバック**
- **コードをブランチ = デザインもブランチ**
- **PRでコードレビュー = 同じPRでデザインdiffもレビュー**

Figmaはクラウド上に独自のバージョン履歴を持つが、Pencilは**Gitが唯一のバージョン管理**。これにより真の「Single Source of Truth」が実現される。

### 完全オープンファイルフォーマット

- 独自ツールでデザインファイルの読み取り・デバッグ・拡張が可能
- ブラックボックスなし、ロックインなし

### Figmaからのインポート

- FigmaからデザインをコピーペーストでインポートⓇ可能
- ベクター、テキスト、スタイルがそのまま移行

### 拡張性（MCPエコシステム）

MCP経由で外部データソースと接続可能：
- データベース、API、チャートデータ
- Playwright / Puppeteer
- その他のエージェント

## ユーザーからのフィードバック

### メリット

- **コンテキストスイッチの排除**：IDE内でスケッチ→生成が完結し、プロトタイピングが大幅に高速化
- **AIトークンの節約**：キャンバスで直接ビジュアル編集すれば、AIトークンを消費しない
- **デザイナー⇔エンジニアの関係変化**：デザイナーが部分的に動く実装を直接作成し、エンジニアに引き継ぐ新しいワークフロー

### 制限事項

- **対応OS**：Mac / Linuxのみ（Windows非対応、ブラウザ版なし）
- **リアルタイムコラボレーション**：なし（Figmaのマルチプレイヤー編集に相当する機能がない）
- **Claude CLI依存**：ローカルでClaude CLIが必要、AI生成には別途Claude Codeサブスクリプションが必要
- **ドキュメント不足**：製品ドキュメントが少ない
- **価格非公開**：料金体系が公開されていない
- **クローズドソース**：内部動作を確認できない

## 競合環境での位置づけ

| ツール | アプローチ |
|--------|-----------|
| **Figma** | クラウドファースト、デザイナー間コラボに最適化 |
| **V0** | FigmaフレームをReactに変換 |
| **Lovable.dev** | フルスタックアプリ生成 |
| **Bolt.new** | ラピッドプロトタイピング |
| **Penpot** | オープンソースFigma代替 |
| **Pencil.dev** | **Git ネイティブ、デザイン=コード統合に最適化** |

Pencilの差別化：**デザインを開発ワークフロー内のバージョン管理可能なアーティファクトとして扱う**唯一のツール。

**トレードオフの本質**：
- Figma → デザイナー同士のコラボレーションに最適
- Pencil → デザインとコードの統合、デザインもコードも行う個人クリエイターに最適

## 結論

Pencil.devは「vibe coding」パラダイムをデザイン領域に拡張するツール。デザインファイルがGitでブランチ・マージできる体験を一度味わうと、クラウド同期型のデザインアーティファクトに戻ることは「Gitを使った後にFTPに戻る」ようなものだと評される。

ハンドオフ問題は、デザインとコードが異なるバージョン管理の下に存在していたから生まれた。Pencilは「もしそうでなかったら？」と問いかける。
